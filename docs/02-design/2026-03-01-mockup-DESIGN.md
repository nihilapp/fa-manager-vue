# Phase 3: UI/UX 구조 및 프로토타입 (Mockup & Screen Flow)

> **목표**: 시스템의 화면 흐름(User Journey)을 설계하고, 재사용 가능한 핵심 UI 컴포넌트를 도출합니다. 설계된 화면 명칭은 Phase 1에서 정의한 용어 사전(Terminology)을 철저히 준수합니다.

## 1. 전반적인 화면 흐름도 (Screen Flow)

- `[ / ]` **로그인 화면 (Login)**
  - 디스코드 ID 입력 폼 제공 (초안 전용 간편 인증).
  - 제출 시 백엔드 조회 후 대시보드로 이동.

- `[ /dashboard ]` **사용자 대시보드 (User Dashboard)**
  - 내 정보 요약 (닉네임).
  - **내 캠페인 목록 (My Campaigns)**: 생성하거나 가입한 캠페인 리스트 표시.
  - 신규 캠페인 생성 버튼 (클릭 시 모달 호출).
  - 특정 캠페인 카드 클릭 시 상세 페이지로 이동.

- `[ /campaigns/[id] ]` **캠페인 상세 페이지 (Campaign Detail)**
  - 상단: 캠페인 이름, 메인 마스터 이름, 기간 정보.
  - 탭 네비게이션: `[ 세션 목록 (Sessions) | 내 캐릭터 (My PC) | 파티원 목록 (Members) ]`
    - **세션 탭**: 회차(Session) 목록 표시, 마스터일 경우 [세션 생성] 버튼 활성화.
    - **캐릭터 탭**: 해당 캠페인에서 내가 생성한 캐릭터 명함(Card)들.
    - **파티원 탭**: `campaign_members` 목록.

- `[ /characters/[id] ]` **캐릭터 상세 페이지 (PC Detail)**
  - 좌측/상단: 캐릭터 명, 멀티 클래스 정보, 레벨, 현재 보유 경험치 및 재화 요약.
  - 본문 본창: 타임라인 뷰어 (Timeline Viewer). 경험치 로그와 자원 변동 로그 최신순 정렬.
  - 마스터 권한일 경우 강제 조정(수정) 버튼 활성화.

- `[ /sessions/[id] ]` **세션 상세 (Session Record)**
  - 세션 참여 캐릭터(`session_characters`) 목록 요약.
  - 세션 진행 시 지급된 골드 및 경험치 보상 기준 표시.

## 2. 와이어프레임 구조 (Layout Wireframe)

### 2.1 글로벌 레이아웃 (Global Layout)
```text
+-------------------------------------------------------------+
| [로고/홈]               (Global Header)      [User Profile] |
+-------------------------------------------------------------+
|                                                             |
|                      Main Content Area                      |
|                                                             |
+-------------------------------------------------------------+
```
> 초기 버전은 관리 복잡도를 줄이기 위해 심플한 Top Navigation Bar(헤더) 구조 하나로만 통일합니다. (사이드바 생략)

### 2.2 캠페인 상세 화면 레이아웃 예시
```text
+-------------------------------------------------------------+
| [ ⬅️ Back ] 🛡️ 캠페인: 왕좌의 탈환           👑 마스터: 홍길동 |
+-------------------------------------------------------------+
| [ 세션 목록 ]  [ 내 캐릭터 ]  [ 참여 멤버 목록 ]            |
+-------------------------------------------------------------+
| +---------------------------------------------------------+ |
| | [ #1 세션: 첫 만남 ]                        [입장/상세] | |

+-------------------------------------------------------------+
```

## 3. 핵심 UI 컴포넌트 도출 (Component Extraction)

와이어프레임을 구현하기 위해 생성해야 할 범용 UI 컴포넌트 목록입니다. (Vue 3 / Tailwind CSS 기반)

1. **Card (카드)**
   - `CampaignCard.vue`: 대시보드의 캠페인 정보 요약 뷰
   - `CharacterCard.vue`: 캐릭터 요약 정보 (이름, 클래스 배열 매핑, 레벨)
   - `SessionCard.vue`: 회차 정보 및 보상치

2. **Modal/Dialog (모달 팝업)**
   - `BaseModal.vue`: 형태만 갖춘 범용 모달 시스템
   - `CreateCampaignModal.vue`: 캠페인 생성용 입력창 (이름, 기간 등)
   - `AddLogModal.vue`: 캐릭터 화면에서 경험치/재화 수동 추가 모달

3. **Timeline (타임라인)**
   - `LogTimeline.vue`: 날짜, 변동치(+/-), 사유(`reason`)를 세로 방향 히스토리로 나열해 주는 컴포넌트. 경험치와 자원 로그에 공통으로 사용.

4. **Form Elements (폼 요소)**
   - `BaseInput.vue`: 텍스트, 숫자 입력을 위한 공통 스타일 랩핑 (UI/UX 통일)
   - `PrimaryButton.vue`, `GhostButton.vue` 등 상태별 버튼.
