# [PLAN] 인증(Auth) API 구현

- **작업일**: 2026-03-02
- **작성자**: Gemini CLI
- **목표**: 비밀번호 없는 디스코드 ID 기반 로그인/로그아웃 기능을 구현하고, UI 세션 유지를 위한 쿠키 처리를 적용한다.

## 1. 📋 요구사항 분석
- **Sign-in**: 디스코드 ID를 받아 시스템 등록 여부를 확인한다. 성공 시 사용자 정보를 쿠키(`auth_user`)에 저장한다.
- **Sign-out**: 저장된 인증 쿠키를 삭제한다.
- **보안**: 별도의 비밀번호는 없으며, 모든 작업은 로그로 기록되므로 추적이 가능하다는 전제를 따른다.
- **응답**: 모든 응답은 `createResponse`를 사용하여 200 OK로 반환한다.

- `X-Discord-ID` 헤더와 연동하여 일관된 인증 경험을 제공한다.
