### 주요 기능 변경
- `GetIcon`이 템플릿 전역 컴포넌트 해석 실패에 영향받지 않도록 변경했다.
- `iconName`처럼 `src/app/utils`에 있는 런타임 유틸이 실제 오토임포트 대상이 되도록 설정을 보강했다.
- `app/utils` 자동 등록과 겹치던 개별 utils 오토임포트 항목은 제거해 중복 경고를 정리했다.
- 푸터, 헤더, 데이터테이블 등 기존 호출처는 수정 없이 동일 인터페이스를 유지한다.
- 최종 Nuxt 빌드 성공으로 컴파일 단계에서 아이콘 컴포넌트 해석 문제와 `iconName` 런타임 누락 문제가 모두 정리됐음을 확인했다.

## 특이 사항
- PowerShell 실행 정책 때문에 `pnpm.ps1` 직접 호출은 실패했다.
- 샌드박스 환경에서는 `spawn EPERM`이 발생해 권한 확장 후 실제 빌드 검증을 수행했다.
- 이후 `common.store.ts`에서 `iconName is not defined` 후속 오류가 확인되어, 개별 심볼 등록 대신 `app/utils` 전체 오토임포트 방식으로 방향을 조정했다.
- `app/utils` 전체 오토임포트 전환 직후 발생한 중복 utils 등록 경고는 기존 개별 등록 항목 제거로 해소했다.
- 빌드 로그에는 별도의 sourcemap 경고와 Node deprecation warning이 남아 있었으나, 이번 장애 해결 범위 밖이다.

## 후속 작업
- 마스터 환경에서 `pnpm.cmd dev` 실행 후 첫 화면 진입 시 `Icon` 관련 경고가 사라졌는지 확인
- 필요하면 sourcemap 경고와 Node deprecation warning을 별도 작업으로 정리
- 보류 중인 `authHelper사용방법분석` 작업 재개 여부 판단
