# 최종 보고서 (Final Report)

## 요약
- 일자: 2026-03-24

- 최종 상태: 완료

## 작업 개요
- 마스터 요청에 따라 Vue Query 중심이던 공통 요청 훅을 Nuxt 4 기본 fetcher 방향으로 전환했다.
- 목표는 `useGet ~ useDelete`라는 진입점은 유지하면서 내부만 `useFetch`, `$fetch` 기반으로 교체하고, GET 요청은 자동 실행 없이 원하는 시점에 실행되도록 만드는 것이었다.

## 결과 요약
### 주요 수정 파일
- `src/app/composables/query/useApiRequest.ts`: 공통 fetch 옵션/에러 추출 유틸 추가
- `src/app/composables/query/useGet.ts`: `useFetch` 기반 수동 실행형으로 전환
- `src/app/composables/query/usePost.ts`, `usePut.ts`, `usePatch.ts`, `useDelete.ts`: `$fetch` 기반 상태 모델로 전환
- `src/app/stores/user.store.ts`: `request.execute()` 명시 호출 방식으로 수정
- `nuxt.config.ts`: Vue Query 관련 설정 잔존 제거
- `src/app/plugins/vue-query.ts`, `src/app/utils/api-client.ts`: 삭제

### 주요 기능 변경
- GET 요청은 더 이상 훅 생성 직후 자동 실행되지 않는다.
- GET은 `const request = await useGet(...); await request.execute();` 방식으로 호출해야 한다.
- mutation 계열은 Vue Query 없이도 `response`, `error`, `pending`, `status`, `execute`를 유지한다.

## 특이 사항
- 실제 프로젝트 내 `useGet` 사용처는 `user.store.ts` 2곳뿐이어서 호출부 수정 범위는 작았다.
- 로컬 의존성 상태 때문에 ESLint와 Nuxt 타입체크는 정상 실행되지 않았다.
- 패키지 의존성 제거까지는 이번 턴에 하지 않아, `package.json`에는 Vue Query/axios 관련 항목이 남아 있을 수 있다.

## 후속 작업

- 브라우저에서 실제 인증 헤더 포함 요청과 에러 리다이렉트 흐름을 수동 확인
