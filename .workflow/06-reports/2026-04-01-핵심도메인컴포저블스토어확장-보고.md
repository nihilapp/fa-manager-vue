# 최종 보고서 (Final Report)

## 요약
- 일자: 2026-04-01
- 작업명: 핵심도메인컴포저블스토어확장
- 최종 상태: 완료

## 작업 개요
- `src/server/api/campaigns`, `src/server/api/characters`, `src/server/api/sessions`에 맞춰 프런트 컴포저블을 순차 대응할 수 있게 정리하고, 상태 소유권을 campaign/character/session/currency 4개 스토어로 고정하는 작업이다.

## 결과 요약
### 주요 수정 파일
- `src/app/composables/query/useDomainResource.ts`: 단건/목록 응답 정규화와 다중 refetch 유틸 추가
- `src/app/stores/campaign.store.ts`: `mine` 축과 멤버/세션/캐릭터 축 추가
- `src/app/stores/session.store.ts`: `mine`, 플레이어, 로그 축 추가
- `src/app/stores/character.store.ts`: 캐릭터 전용 스토어 신규 생성
- `src/app/stores/currency.store.ts`: 화폐 현황 전용 스토어 신규 생성
- `src/app/composables/campaigns/useCampaigns.ts`: campaigns API 대응 컴포저블 묶음 추가
- `src/app/composables/characters/useCharacters.ts`: characters API 대응 컴포저블 묶음 추가
- `src/app/composables/sessions/useSessions.ts`: sessions API 대응 컴포저블 묶음 추가

### 주요 기능 변경
- campaigns/characters/sessions API를 프런트에서 경로 단위로 감쌀 수 있는 컴포저블 표면을 만들었다.
- 캠페인 관련 상태는 campaign 스토어, 세션 관련 상태는 session 스토어 안에서 직접 관리하도록 기준선을 고정했다.
- 서버 응답이 불안정한 경로를 프런트 공통 유틸에서 정규화해 스토어 저장이 가능하도록 만들었다.

## 특이 사항
- 서버의 여러 상세/CUD 라우트가 실제로는 목록 응답을 반환하고 있어, 프런트에서 `extractItemById`와 fallback 정규화를 사용했다.
- `SectionPage.vue`는 기존 작업 트리에 이미 수정 사항이 있었고, 이번 작업에서는 건드리지 않았다.
