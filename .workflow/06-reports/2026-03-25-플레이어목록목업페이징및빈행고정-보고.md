## 작업 개요
- 플레이어 목록 화면에서 실제 API 데이터를 잠시 걷어내고, 충분한 양의 목업 데이터로 페이징과 테이블 밀도를 테스트할 수 있게 만드는 작업이다.
- 동시에 마지막 페이지에서 데이터 수가 적어도 테이블 높이가 줄어들지 않도록 빈 행을 고정적으로 보여주게 조정했다.

### 주요 기능 변경
- 플레이어 목록이 25건 목업 데이터로 페이징되며, 페이지 이동 테스트가 가능해졌다.
- 마지막 페이지처럼 데이터가 부족한 경우에도 `pageSize`만큼 빈 행이 표시되어 테이블 높이가 고정된다.
- 플레이어 페이지에서 바깥 배경 래퍼를 제거해 테이블 자체 배경과 충돌하지 않게 했다.

## 특이 사항
- `context7 MCP`는 현재 세션에서 제공되지 않아 로컬 코드와 문서 기준으로 작업했다.
- 실제 API 연결은 테스트 목적상 의도적으로 제거된 상태다.

## 후속 작업
- 테스트가 끝나면 실제 `useGetUserList()` 연동으로 복귀하거나, 목업/실데이터 전환 플래그를 분리
- 필요 시 placeholder row에 더 자연스러운 skeleton 또는 점선 placeholder 표현 추가
