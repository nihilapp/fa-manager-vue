## 작업 개요
- `src/app/types/common.types.ts`에서 `DataTableColumn`이 제네릭으로 변경된 뒤, 수동 오토임포트 선언과 컬럼 관련 유틸/컴포넌트 타입 시그니처가 뒤따르지 못한 부분을 정리했다.

## 결과 요약
### 주요 수정 파일
- `src/app/types/manual-app-imports.d.ts`: 전역 `DataTableColumn<T>` 타입 별칭으로 수정
- `src/app/composables/useColumns.ts`: 제네릭 컬럼 배열 반환으로 수정
- `src/app/utils/column-builder.ts`: 제네릭 컬럼 빌더로 수정
- `src/app/components/table/DataTable.vue`: 내부 헬퍼를 `DataTableColumn<T>` 기준으로 수정하고 안전한 셀 값 접근 추가
- `src/app/components/players/PlayerListTable.vue`: 행 타입을 명시해 컬럼 배열 제네릭 추론 충돌 해결

### 주요 기능 변경
- 전역 오토임포트 타입에서 `DataTableColumn<T>`를 직접 사용할 수 있게 됐다.
- 컬럼 빌더와 컬럼 배열 유틸이 행 타입의 `keyof T` 정보를 유지한다.
- 자유 문자열 key를 허용하는 구조에서도 `DataTable.vue`가 안전하게 값을 읽도록 보정됐다.
