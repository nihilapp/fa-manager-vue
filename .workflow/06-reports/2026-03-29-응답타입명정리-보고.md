# 최종 보고서 (Final Report)

## 요약
- 일자: 2026-03-29
- 작업명: 응답타입명정리
- 최종 상태: 완료

## 작업 개요
- `src/app`, `src/server` 양쪽에서 응답 타입 기준 명칭을 `BaseResponse` 계열에서 `BaseApiResponse`로 통일하는 작업이다.
- 마스터 요청에 따라 앱의 `ApiErrorResponse`도 별도 구조가 아니라 `BaseApiResponse<null>` 별칭으로 정리했다.

## 결과 요약
### 주요 수정 파일
- `src/server/types/response.types.ts`: 공용 응답 인터페이스를 `BaseApiResponse<T>`로 변경하고 `responseTime`을 공용 필드로 포함했다. `data`는 `T`를 유지한다.
- `src/server/utils/base-response.ts`: 서버 응답 헬퍼 클래스를 `BaseApiResponse`로 전환했다.
- `src/server/types/manual-server-imports.d.ts`: 서버 전역 타입 오토임포트를 `BaseApiResponse` 기준으로 정리했다.
- `src/server/api/**`, `src/server/utils/auth.ts`, `src/server/utils/access.ts`: 서버 사용처 전반의 호출과 타입 주석을 새 명칭으로 치환했다.
- `src/app/composables/query/useApiRequest.ts`: 앱 응답 타입을 서버 `BaseApiResponse` 기반으로 맞추고 `ApiErrorResponse = BaseApiResponse<null>`로 정리했다.
- `src/app/types/manual-app-imports.d.ts`: 앱 전역 타입 오토임포트를 서버 공용 타입 기준으로 통일했다.

### 주요 기능 변경
- 프론트와 백엔드가 같은 `BaseApiResponse` 계약을 기준으로 응답을 해석한다.
- 서버 응답 생성 헬퍼 호출이 전부 `BaseApiResponse.data/page/error`로 통일됐다.
- 앱 에러 응답 타입이 독립 유니온이 아니라 `BaseApiResponse<null>` 별칭으로 단순화됐다.
- 성공 응답이 `null`일 수 있는 경우는 인터페이스가 아니라 제네릭 인자 자체로 표현한다.
- 앱 타입 export와 서버 Nitro 값 오토임포트의 이름 충돌을 제거했다.
