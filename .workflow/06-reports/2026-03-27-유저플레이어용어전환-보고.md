# 최종 보고서 (Report)

## 작업 개요

- 작업명: 유저플레이어용어전환
- 목표: `users` 기반 명칭과 API 계약을 `players` 기준으로 전환한다.
- 범위: 프런트 식별자, 서버 API, DTO/자동임포트, Drizzle 스키마, PRD 문서

## 결과 요약

- 프런트
  - `usePlayerStore`, `PlayerListTable`, `useGetPlayerList`, `useGetMyInfo`, `queryKeys.players` 기준으로 전환했다.
  - `/players` 페이지와 헤더의 플레이어 정보 조회 연결을 새 명칭으로 교체했다.
- 서버
  - `/api/users` 계열을 `/api/players`로 이동했다.
  - `player.dto.ts`, `Player*Dto`, `PlayerRole`, `PlayerStatus`, `playersTable` 기준으로 핵심 명칭을 재정렬했다.
  - 응답 메시지에 플레이어 기준 상수를 추가하고 기존 사용자 상수는 호환 별칭으로 유지했다.
- 문서
  - `PRD/Api-Specification.md`, `PRD/Task-List.md`의 주요 API 경로와 용어를 플레이어 기준으로 갱신했다.

## 특이사항

- `pnpm.cmd exec nuxi typecheck`는 `vue-tsc` 다운로드 단계에서 네트워크/권한 문제로 실패했다.
- DB 실제 테이블명과 인덱스명은 코드 기준으로 `players`로 바뀌었지만, 운영 스키마 반영 여부는 별도 확인이 필요하다.
- relation 필드명과 일부 컬럼명(`user`, `userId`)은 이번 작업에서 전면 치환하지 못했다.

## 후속 작업

- `vue-tsc`가 로컬에 준비된 환경에서 `nuxi typecheck`와 실제 빌드를 재실행한다.
- Drizzle 마이그레이션 또는 DB 반영 절차로 `users` -> `players` 물리 스키마 변경을 검증한다.
- relation 필드명과 DTO 내부 속성명(`user`, `userId`)까지 플레이어 기준으로 추가 정리한다.
