# 최종 보고서

## 작업 개요
- 작업명: 디스코드식별쿠키지속화
- 작업 기간: 2026-03-23
- 목적: 최초 진입 시 `x-discord-id` 헤더로 식별한 사용자를 쿠키에 저장하고, 새로고침 및 후속 API 요청에서도 동일 사용자를 복원할 수 있도록 인증 흐름을 개선한다.

## 결과 요약
- `authHelper(event)`에 `헤더 > 쿠키 > 개발 환경 ADMIN_DISCORD_ID` 우선순위를 적용했다.
- 헤더로 식별된 사용자는 `discord_id` `httpOnly` 쿠키에 저장되도록 변경했다.
- 헤더가 없는 후속 요청은 쿠키 값을 fallback으로 사용해 동일 사용자를 조회할 수 있게 했다.
- 쿠키 값이 더 이상 유효하지 않으면 stale 쿠키를 자동 삭제하도록 정리했다.

## 주요 수정 파일
- `src/server/utils/auth.ts`

## 작업 중 확인된 특이사항
- 프런트 Axios는 `x-discord-id`를 자동 전달하지 않으므로, 서버 인증 유틸에서 쿠키 지속화를 처리하는 방식이 적합했다.
- 개발 환경 fallback은 유지하되, 실제 사용자 식별은 헤더와 쿠키가 우선하도록 재정렬했다.
