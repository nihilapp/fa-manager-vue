# 분석 (Analysis)

## 요약
- 요청 내용: `Button.vue`에서 `fill`은 기본 색상을 500 단계로 두고 hover 시 600 단계로 더 진하게 변경, `outline`은 평상시 배경 없이 표시하되 hover 시 테두리와 동일한 색상 배경이 올라오도록 수정
- 범위: 공용 버튼 컴포넌트의 variant 클래스 조정
- 구현 가능성: 가능

## 요구사항 요약
- `fill` 모드의 색상 기준을 전 색상군에 대해 "기본 500, hover 600"으로 통일해야 한다.
- `outline` 모드는 기본 상태에서 흰 배경 또는 무배경 기반을 유지하되, hover 시 배경색이 현재 테두리색과 동일 계열로 채워져야 한다.
- 공용 버튼 컴포넌트이므로 `gray`, `red`, `blue`, `black` 전 색상 조합에 일관되게 반영되어야 한다.

## 현재 상태
### 관련 파일
- `src/app/components/common/Button.vue`: `cva` compoundVariants로 `mode`/`color`별 버튼 스타일을 정의한다. 현재 `fill`은 hover 시 더 연한 색상으로 바뀌는 조합이 포함되어 있고, `outline`은 hover 시 약한 tint 배경만 적용된다.
- `src/app/assets/styles/tailwind.css`: 프로젝트에서 사용하는 Tailwind 테마 토큰과 기본 유틸리티 동작 기준을 정의한다. `text-sm`, `rounded-2`, `font-500` 등 커스텀 토큰 사용 여부를 판단할 수 있다.
- `PRD/Coding-Rules.md`: 공용 컴포넌트 수정 시 기존 네이밍 및 이벤트 핸들러 규칙을 유지해야 하는 기준 문서다.

### 현재 로직 분석
- `Button.vue`는 `buttonVariants`의 `compoundVariants`에서 실제 색상 클래스를 결정한다.
- `fill` 모드:
  - `gray`는 기본/hover 모두 `stone-700`과 `stone-600`을 사용해 hover가 더 밝아진다.
  - `red`, `blue`는 기본 `600`, hover `500`으로 역시 hover가 더 밝아진다.
  - `black`은 `900`에서 `700`으로 바뀌어 hover가 더 밝아진다.
- `outline` 모드:
  - 기본 상태에 `bg-white`가 지정되어 있고, hover 시에도 `*-50` 또는 `black-100` 수준의 옅은 배경만 적용된다.
  - 마스터 요구사항의 "테두리와 같은 색으로 배경이 올라와야 한다"와 차이가 있다.

## 영향도 평가
### 직접 영향
- `Button.vue` 내부 `compoundVariants`의 색상 클래스 문자열이 직접 수정 대상이다.
- 버튼의 hover 대비와 텍스트 가독성을 함께 맞춰야 하므로 `outline` hover 시 텍스트 색상도 배경 대비에 맞게 조정할 가능성이 있다.

### 간접 영향
- `Button.vue`를 사용하는 모든 화면에서 hover 시각 스타일이 즉시 변경된다.
- 스냅샷 테스트나 시각 회귀 테스트가 있다면 영향 받을 수 있으나, 현재 확인 범위에서는 관련 테스트 코드는 아직 확인하지 못했다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- 프로젝트 색상 토큰이 일반 Tailwind 기본 팔레트와 일부 다를 수 있어 `gray`/`black` 계열에서 500/600 대응값을 기존 토큰 체계 안에서 해석해야 한다.
- `outline` hover 배경을 테두리색과 동일하게 올릴 경우 텍스트 가독성을 위해 hover 텍스트 색상도 함께 재조정해야 한다.

### 가정 / 불확실성
- 마스터가 언급한 "500/600" 기준은 `fill` 배경색 기준으로 해석했다.
- `outline`의 기본 배경은 실질적으로 무배경처럼 보이는 `bg-white` 유지로 해석했으며, hover 시에는 테두리와 동일 계열 배경색으로 채우는 방향이 적절하다.
- `gray`와 `black`은 기존 토큰 체계상 500/600이 의미하는 시각적 단계가 일반 색상군과 완전히 동일하지 않을 수 있어 설계 단계에서 보정 기준을 명시해야 한다.
