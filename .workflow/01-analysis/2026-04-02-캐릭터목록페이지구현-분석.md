# 분석 (Analysis)

## 요약
- 요청 내용: `src/app/pages/characters/index.vue`를 플레이어 목록 페이지 패턴으로 구현하고, 이름/클래스/현재 레벨/현재 경험치/자금/비욘드 링크 컬럼을 보여준다.
- 범위: 캐릭터 목록 페이지, 목록 컨테이너 컴포넌트, 테이블 컴포넌트, 상단 생성 버튼
- 구현 가능성: 가능

## 요구사항 요약
- 플레이어 목록 페이지처럼 `SectionPage` 안에 리스트와 상단 액션 버튼을 둔다.
- 목록 컬럼은 이름, 멀티클래스 포함 클래스 문자열, 현재 레벨, 현재 경험치, 자금, 비욘드 링크를 제공한다.
- 상단 버튼 레이블은 `캐릭터 생성`이다.

## 현재 상태
### 관련 파일
- `src/app/pages/players/index.vue`
  - 메타 설정 후 `PlayerList` 컴포넌트를 렌더링하는 페이지 래퍼다.
- `src/app/components/players/PlayerList.vue`
  - 페이지네이션과 버튼 이벤트를 관리하는 목록 컨테이너다.
- `src/app/components/players/PlayerListTable.vue`
  - `DataTable` 기반 목록 컬럼 및 액션 렌더링 기준 컴포넌트다.
- `src/app/pages/characters/index.vue`
  - 현재 비어 있다.
- `src/app/composables/characters/useGetCharacterList.ts`
  - `/characters` 목록을 조회해 `characterList`, `characterPageData` 축에 저장한다.
- `src/app/stores/character.store.ts`
  - `characterList`, `characterPageData`, `myCharacterList` 등 캐릭터 목록 상태 축을 제공한다.
- `src/server/types/dto/character.dto.ts`
  - 목록 렌더링에 필요한 `name`, `currentLevel`, `currentExp`, `currentCurrencyGp`, `classes` 필드를 확인할 수 있다.

### 현재 로직 분석
- 캐릭터 목록 조회와 저장은 이미 구현되어 있어 화면 연결만 하면 된다.
- 플레이어 목록 패턴은 `페이지 -> 목록 컨테이너 -> 목록 테이블` 3단 구조다.
- DTO와 코드 검색 기준으로 비욘드 링크 필드는 현재 응답 구조에 없다.

## 영향도 평가
### 직접 영향
- `src/app/pages/characters/index.vue`
- `src/app/components/characters/CharacterList.vue` 신규 추가
- `src/app/components/characters/CharacterListTable.vue` 신규 추가

### 간접 영향
- 캐릭터 생성 페이지 경로가 아직 없으면 상단 버튼은 임시 경로 또는 후속 작업 대상으로 남을 수 있다.
- 비욘드 링크 컬럼은 실제 DTO 필드가 생기면 후속 연결이 필요하다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- 멀티클래스 문자열을 표에서 간결하게 보여 주도록 포맷팅이 필요하다.
- 비욘드 링크 필드가 현재 DTO에 없으므로 안전한 fallback이 필요하다.

### 가정 / 불확실성
- 비욘드 링크는 현재 데이터 계약 부재로 `-`를 표시한다.
- 상세/수정 액션은 이번 요청에 포함되지 않았으므로 추가하지 않는다.
