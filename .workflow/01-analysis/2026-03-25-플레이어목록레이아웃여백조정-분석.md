# 분석 (Analysis)

## 요약
- 요청 내용: 플레이어 목록 테이블이 페이지 전체 폭을 과하게 차지하지 않도록 여백과 폭 제한을 다시 부여
- 범위: `src/app/pages/players/index.vue`
- 구현 가능성: 가능

## 요구사항 요약
- 테이블은 독자적인 카드형 배경을 유지하되, 페이지 가장자리까지 꽉 차 보이면 안 된다.
- 이전에 제거한 배경 섹션은 복원하지 않고, 여백과 최대 폭만 담당하는 중립 래퍼가 필요하다.
- 즉 배경 없는 레이아웃 컨테이너를 사용해 시각적 숨 쉴 공간을 만들어야 한다.

## 현재 상태
### 관련 파일
- `src/app/pages/players/index.vue`: 현재 `DataTable`을 루트에서 직접 렌더링해 페이지 폭을 거의 그대로 차지한다.
- `src/app/components/table/DataTable.vue`: 자체 카드형 배경과 shadow를 가지고 있어 배경을 추가로 감쌀 필요는 없지만, 부모가 여백을 주지 않으면 화면을 꽉 채워 보인다.

### 현재 로직 분석
- 현재 문제는 데이터나 테이블 컴포넌트 자체가 아니라 페이지 레이아웃 레벨의 여백 부재다.
- 따라서 `index.vue` 쪽에 `max-width`, `margin: auto`, 적당한 `padding`만 가진 래퍼를 두면 해결 가능하다.
- 배경 색이나 그라디언트가 없는 래퍼라면 이전 지적과 충돌하지 않는다.

## 영향도 평가
### 직접 영향
- `src/app/pages/players/index.vue`의 최상위 템플릿 구조 조정

### 간접 영향
- 공통 `DataTable.vue`는 그대로 유지되므로 다른 목록 화면에는 영향이 없다.

## 실행 판단

- 가능

### 가정 / 불확실성
- `context7 MCP`는 현재 세션에서 제공되지 않아 로컬 코드와 문서 기준으로 분석을 수행했다.
- 마스터 의도는 배경 복원이 아니라 레이아웃 spacing 복원으로 해석했다.
