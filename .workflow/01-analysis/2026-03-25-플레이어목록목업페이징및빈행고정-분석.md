# 분석 (Analysis)

## 요약
- 요청 내용: 플레이어 목록 페이지의 실제 API 조회를 목업 데이터 25건 기반으로 전환하고, 테이블은 항상 `pageSize` 개수만큼 행을 보여주며 페이지 래퍼 배경을 제거
- 범위: `src/app/pages/players/index.vue`, `src/app/components/table/DataTable.vue`, 필요 시 `src/app/components/table/Paginator.vue`
- 구현 가능성: 가능

## 요구사항 요약
- 현재 `useGetUserList()` 기반 실제 데이터 조회를 잠시 제거하고, 화면 테스트를 위한 목업 플레이어 데이터 25건 정도를 페이지 내부에서 사용해야 한다.
- 페이징은 목업 데이터 기준으로 실제처럼 동작해야 하며, 데이터가 많은 상태에서 페이지 이동 테스트가 가능해야 한다.
- 테이블은 현재 페이지 데이터 개수가 `pageSize`보다 적더라도 항상 `pageSize` 개수만큼 행을 보여야 한다. 부족한 수만큼 빈 행을 렌더링해 높이를 고정해야 한다.
- 플레이어 페이지는 바깥 `section` 배경을 제거하고 테이블 자체만 배치해야 한다.

## 현재 상태
### 관련 파일
- `src/app/pages/players/index.vue`: 현재 `useGetUserList().execute()`를 사용해 실제 데이터를 가져오고, 테이블 바깥에 별도 배경 섹션을 두고 있다.
- `src/app/components/table/DataTable.vue`: 전달받은 `items`만큼만 행을 렌더링하며, 마지막 페이지에서 데이터가 적으면 테이블 높이가 줄어든다.
- `src/app/components/table/Paginator.vue`: 전달받은 `pagination` 구조를 기반으로 페이지 정보를 표시하며, 현재 구조라면 목업 pagination 객체만 맞춰주면 그대로 재사용 가능하다.

### 현재 로직 분석
- `players/index.vue`는 서버 조회 결과를 `userStore`에서 읽어 바로 테이블에 전달한다.
- 현재 `DataTable.vue`는 `items.length === 0`일 때만 empty state를 보여주고, 일부만 채워진 마지막 페이지의 빈 공간을 별도 행으로 채우지 않는다.
- 페이지 크기 기준은 `pagination.pageSize`에 이미 포함되어 있으므로, 빈 행 개수는 `pageSize - items.length`로 계산 가능하다.
- `Paginator.vue`는 `currentPage`, `pageSize`, `totalPages`, `filteredElements` 등을 쓰므로, 플레이어 페이지에서 이 구조의 목업 pagination 객체를 제공하면 실제 API 없이도 동작할 수 있다.

## 영향도 평가
### 직접 영향
- `src/app/pages/players/index.vue`의 데이터 소스, 페이지 상태, 페이징 계산, 배경 래퍼 구조 변경
- `src/app/components/table/DataTable.vue`의 행 렌더링 로직 변경

### 간접 영향
- `DataTable.vue`의 빈 행 렌더링은 다른 페이지에서도 `pagination.pageSize`가 제공될 경우 동일하게 적용될 수 있다.
- 현재 플레이어 목록은 목업 데이터로 바뀌므로 실제 API 연동 확인은 일시적으로 보류된다.
- `Paginator.vue`는 직접 수정이 필요 없을 가능성이 높지만, 페이지 정보가 목업 pagination과 자연스럽게 맞는지 확인은 필요하다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- 빈 행은 실제 데이터 행과 시각적으로 섞이지 않으면서도 높이는 동일하게 유지해야 한다.
- 목업 pagination 구조를 현재 `ListPageData` 계약과 맞추되, 마지막 페이지의 데이터 수가 적을 때 정보 라벨이 자연스럽게 보여야 한다.

### 가정 / 불확실성
- `context7 MCP`는 현재 세션에서 제공되지 않아 로컬 코드와 문서 기준으로 분석을 수행했다.
- 이번 작업은 테스트 목적의 임시 목업 전환으로 이해하고 있으며, 실제 API 연동 복귀는 후속 작업으로 남는다.
