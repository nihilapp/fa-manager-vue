## 요구사항 재해석
- `src/app`의 `useSetMeta` 유틸이 현재 구조와 타입 기준에서 올바르게 동작하는지 검토하고 보정해야 합니다.
- `useGet`부터 `useDelete`까지 쿼리 컴포저블이 인증 헤더, 성공/에러 콜백, 실행 제어를 일관되게 처리해야 합니다.
- `src/server`처럼 앱 레이어도 명시적 오토임포트 설정을 적용해 수동 선언 파일 의존도를 줄여야 합니다.

## 현황 파악
- `nuxt.config.ts`에서 앱 `imports.autoImport`가 `false`로 비활성화되어 있습니다.
- `src/app/types/manual-app-imports.d.ts`가 Vue, Nuxt, Pinia 전역 심볼을 수동 선언하고 있으나 실제 런타임 오토임포트는 제공하지 않습니다.
- `src/app/composables/query/*`는 `POST`, `PUT`, `PATCH`, `DELETE` 로직이 거의 동일한데 파일별로 중복 구현되어 있습니다.
- `useGet`은 `enabled` 반응형 값을 `watch`와 수동 `execute()`로 동시에 처리해 중복 실행 가능성이 있습니다.
- `src/app` 내부 import는 `~/config`, `~/types`, `~/utils`를 사용하지만 실제 파일은 `src/app/*`, `src/server/types/*`에 있어 경로 계층이 맞지 않습니다.
- `src/app/types/common.types.ts`에는 `ApiResponse`, `PageDataType`, `ResponseCode` 정의가 없어 현재 참조와 타입 구조가 불완전합니다.

## 영향 파일 및 역할
- `nuxt.config.ts`: 앱/서버 오토임포트 정책과 사용자 정의 preset/import를 선언하는 핵심 설정 파일입니다.
- `src/app/composables/useSetMeta.ts`: 페이지 메타데이터와 canonical/OG/Twitter 값을 조합하는 유틸입니다.
- `src/app/composables/query/useGet.ts`: 조회성 요청과 지연 실행 제어를 담당합니다.
- `src/app/composables/query/usePost.ts`, `usePut.ts`, `usePatch.ts`, `useDelete.ts`: 변경성 요청의 실행 상태와 응답 처리 로직을 담당합니다.
- `src/app/utils/api-error-handler.ts`: 공통 에러 판정과 리다이렉트/로깅 정책을 담고 있습니다.
- `src/app/types/common.types.ts`: 앱에서 사용하는 메타 타입과 API 응답 타입의 기준 파일입니다.
- `src/app/types/manual-app-imports.d.ts`: 앱 전역 심볼 선언 파일이며 오토임포트 활성화 이후 역할 조정이 필요합니다.
- `src/app/stores/user.store.ts`: `useGet`과 공유 DTO 타입을 실제로 사용하는 대표 호출 지점입니다.

## 영향도 평가
- 직접 영향: 앱 전역 오토임포트, 메타 유틸, 쿼리 컴포저블의 실행 방식과 타입 안정성이 바뀝니다.
- 간접 영향: 기존 `src/app` 내 잘못된 import 경로가 정리되며, 새 컴포저블 사용 시 명시 import 부담이 줄어듭니다.

## 불확실성
- `.workflow/template` 폴더는 현재 저장소에 없어 기존 산출물 형식을 기준으로 문서를 직접 작성해야 합니다.
- 일부 앱 코드는 아직 사용처가 제한적이라 쿼리 컴포저블의 외부 호출 패턴이 충분히 드러나지 않았습니다.
- `context7 MCP` 검색 지침이 있으나 현재 세션에서는 해당 도구가 제공되지 않아 로컬 코드 기준 분석으로 대체합니다.

## 실행 판단
- 쿼리 컴포저블은 공통 헬퍼를 두고 `GET`과 mutation 계열을 분리해 중복과 실행 타이밍 문제를 제거하는 것이 적절합니다.
- 앱 오토임포트는 `nuxt.config.ts`에서 재활성화하고, `src/server`와 유사하게 명시적 import/preset 목록을 등록하는 것이 적절합니다.
- 수동 선언 파일은 중복 전역 선언을 제거하고 최소한의 안내용 파일로 축소하는 것이 적절합니다.
