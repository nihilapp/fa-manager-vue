# 분석 (Analysis)

## 요약
- 요청 내용: `src/app/components/table/DataTable.vue`에서 굳이 필요 없는 변수를 제거하고, 유지가 필요한 변수만 남기도록 정리한다.
- 범위: `DataTable.vue`의 페이지네이션 관련 계산 변수, 컬럼 보조 함수, `Paginator.vue`와의 스코프 계약
- 구현 가능성: 가능

## 요구사항 요약
- 마스터 요청은 기능 추가가 아니라 기존 컴포넌트 내부 변수 정리다.
- 제거 기준은 "선언만 있고 실제 렌더링 또는 이벤트 처리에 기여하지 않는 변수"여야 한다.
- 단, `Paginator.vue`는 별도 props 없이 상위 `DataTable.vue` 스코프의 식별자를 직접 사용하므로, 상위 템플릿에서 직접 보이지 않아도 하위 컴포넌트 템플릿에서 소비되는 값은 유지 대상이다.

## 현재 상태
### 관련 파일
- `src/app/components/table/DataTable.vue`: 테이블 렌더링과 페이지네이션 상태 계산을 담당하는 상위 컴포넌트다. `normalizedPagination`에서 계산한 값을 다수의 computed로 다시 펼쳐서 템플릿 및 `Paginator.vue`에서 사용한다.
- `src/app/components/table/Paginator.vue`: 별도 props/emit 선언 없이 `DataTable.vue`의 스코프에 접근해 페이지네이터 UI를 렌더링한다. `filteredElements`, `startItem`, `endItem`, `hasPrev`, `hasNext`, `pageNumbers`, `currentPage`, `goToPage`, `goToFirstPage`, `goToPrevPage`, `goToNextPage`, `goToLastPage`, `paginationEnabled`에 의존한다.
- `src/app/types/common.types.ts`: `DataTableColumn` 타입을 정의한다. `DataTable.vue`의 컬럼 정렬, 폭 계산 함수가 이 타입을 기준으로 동작한다.

### 현재 로직 분석
- `normalizedPagination`은 외부에서 받은 `pagination`을 안전한 페이지네이션 구조로 정규화한다.
- 이후 `filteredElements`, `currentPage`, `pageSize`, `totalPages`, `firstPage`, `lastPage`, `hasPrev`, `hasNext`, `prevPage`, `nextPage`를 개별 computed로 다시 분해한다.
- 이 중 `pageSize`, `prevPage`, `nextPage`, `firstPage`, `lastPage`, `currentPage`, `filteredElements`, `hasPrev`, `hasNext`, `totalPages`는 다시 `startItem`, `endItem`, `pageNumbers`, `goToPage`, `goToPrevPage`, `goToNextPage`, `goToFirstPage`, `goToLastPage`, `Paginator.vue` 템플릿에서 소비된다.
- `pageGroupStart`는 `pageNumbers` 계산을 위한 중간 변수다.
- 현 시점에서 "선언만 있고 실제로 전혀 소비되지 않는 페이지네이션 변수"는 확인되지 않았다.
- 다만 `normalizedPagination.value`를 반복 참조하기 위해 계산값을 과도하게 분해한 구조라, 일부는 별도 computed를 없애고 직접 `normalizedPagination.value?.xxx` 또는 단일 구조 객체 참조로 축약할 수 있다.

## 영향도 평가
### 직접 영향
- `src/app/components/table/DataTable.vue`의 스크립트 구조 단순화
- `src/app/components/table/Paginator.vue`가 참조하는 식별자 집합 유지 여부 확인 필요

### 간접 영향
- `Paginator.vue`가 암묵적 상위 스코프 계약에 의존하므로, 변수명을 함부로 제거하거나 변경하면 페이지네이터 렌더링이 즉시 깨질 수 있다.
- 페이지네이션 관련 이벤트 핸들러와 현재 페이지 버튼 그룹 계산이 연결되어 있어, 중간 변수를 합치는 과정에서 경계값 처리(`firstPage`, `lastPage`)가 훼손되지 않도록 주의해야 한다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- 겉보기에는 미사용 변수처럼 보여도 `Paginator.vue`가 직접 참조하는 값이 많아 오판 가능성이 있다.
- 안전한 정리 방향은 "완전 제거"보다 "중복 파생 변수 축소"에 가깝다.

### 가정 / 불확실성
- 현재 구조상 `Paginator.vue`는 상위 스코프 식별자를 직접 참조하는 Vue SFC 컴파일 동작을 전제로 한다.
- `context7 MCP`는 현재 세션에서 사용할 수 없어 로컬 코드 기준으로 분석했다.
