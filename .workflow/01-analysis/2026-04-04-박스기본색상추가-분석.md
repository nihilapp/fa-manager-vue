# 분석 (Analysis)

## 요약
- 요청 내용: `Box.color`에 `default`를 추가하고 기본 동작을 `배경 없음 + border-slate-200`으로 변경한다
- 범위: `src/app/components/common/Box.vue`
- 구현 가능성: 가능

## 요구사항 요약
- `Box.color`는 앞으로 `default` 값을 지원해야 한다
- `default`는 테두리만 `border-slate-200`을 적용하고 배경색은 없어야 한다
- 현재 기본 색상이 `gray`로 설정되어 있어 색을 지정하지 않은 `Box`가 옅은 stone 배경을 갖는데, 이를 neutral한 기본값으로 바꿔야 한다

## 현재 상태
### 관련 파일
- `src/app/components/common/Box.vue`
  - 현재 `color?: StatusColor`를 사용하며 defaultVariants의 `color`는 `gray`다
  - `gray`는 `border-stone-200 bg-stone-50`로 정의되어 있다
- `src/app/types/common.types.ts`
  - `StatusColor`는 실제 색상 이름만 포함한다

### 현재 로직 분석
- 현재 `Box`는 색상을 주지 않으면 `gray`가 적용되어 기본 배경색이 생긴다
- 마스터 요청은 `default`를 만들어 명시적 기본 상태를 분리하는 것이다
- `default`는 실색상 개념이 아니므로 `StatusColor` 자체에 추가하기보다 `Box` 내부에서만 확장 타입으로 다루는 편이 공통 타입 오염을 줄인다

## 영향도 평가
### 직접 영향
- `src/app/components/common/Box.vue`

### 간접 영향
- 현재 `color`를 주지 않는 모든 `Box`는 배경색이 없는 형태로 바뀐다
- `color=\"gray\"`를 명시한 박스는 기존처럼 stone 배경을 유지하므로, 기존 의도를 잃지 않는다

- 가능

### 예상 기술 난제
- 없음
