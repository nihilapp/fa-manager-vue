## 요약

- 범위: 기본 레이아웃과 그 내부 공통 컨텐츠 래핑 구조
- 구현 가능성: 가능

## 요구사항 요약
- 모바일/태블릿 구간은 기존처럼 전체 폭을 유지한다.
- PC 구간에서는 메인 슬롯 영역을 가운데 정렬하고 최대 너비를 1200px로 제한한다.
- 요청 지점은 `default.vue`이므로 공통 레이아웃 컴포넌트의 전역 동작 변경보다 해당 레이아웃 내부에서 제어하는 편이 안전하다.

## 현재 상태
### 관련 파일
- `src/app/layouts/default.vue`: `CommonHeader`, `CommonContent`, `CommonFooter`를 조합하는 기본 레이아웃이다. 현재 슬롯 래퍼가 없어 메인 컨텐츠가 전체 폭을 그대로 사용한다.
- `src/app/components/layout/common-layout/CommonContent.vue`: 단순히 `<main>`과 슬롯만 제공한다. 여기서 폭을 제한하면 다른 사용처까지 영향이 갈 수 있다.
- `src/app/assets/styles/tailwind.css`: 커스텀 브레이크포인트와 spacing 스케일을 정의한다. `--spacing: 0.25rem` 기준으로 `max-w-300`은 75rem, 즉 1200px에 해당한다.

### 현재 로직 분석
- `default.vue`는 `CommonContent` 안에 슬롯을 직접 렌더링하므로, 메인 폭 제한은 `CommonContent` 내부가 아니라 `default.vue` 슬롯 래퍼에서 제어할 수 있다.
- 프로젝트 브레이크포인트는 `lg: 1024px`부터 PC 구간으로 해석할 수 있다.
- 따라서 `w-full lg:max-w-300 mx-auto` 패턴을 사용하면 모바일은 전체 폭, PC는 최대 1200px 정렬이 가능하다.

## 영향도 평가
### 직접 영향
- `default.vue`를 사용하는 페이지들의 메인 컨텐츠 폭이 PC에서 1200px로 제한된다.
- 헤더와 푸터는 현재 구조를 유지하고, 슬롯 본문만 제한된다.

### 간접 영향
- 각 페이지가 이미 자체 `max-w-*` 래퍼를 가지고 있으면 실제 표시 폭은 더 좁을 수 있다.
- `CommonContent.vue`를 수정하지 않으므로 다른 레이아웃이나 별도 사용처에는 영향이 없다.
