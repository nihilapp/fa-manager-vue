## 요약
- 요청 내용: `src/app`, `src/server` 전역에서 함수 선언 스타일을 점검하고, `.vue` 파일은 화살표 함수, 그 외 파일은 함수 선언식으로 강제하는 작업의 사전 보고서를 작성
- 범위: Vue SFC, app composable/store/util, server util/api 내부 지역 함수 선언
- 구현 가능성: 가능

## 요구사항 요약
- Vue SFC 내부에서 선언된 명명 함수는 모두 화살표 함수로 스와핑해야 한다.
- `.vue`가 아닌 `src/app`, `src/server` 파일에서 `const fn = () => {}` 형태로 선언된 함수는 함수 선언식으로 스와핑해야 한다.
- 실제 스와핑에 들어가기 전, 위반 지점을 파일 단위로 집계하고 변경 위험을 분석한 보고서가 먼저 필요하다.
- 이번 단계에서는 자동 수정 없이 점검 결과와 후속 작업 기준만 확정한다.

## 현재 상태
### 관련 파일
- `src/app/components/table/DataTable.vue`: 테이블 렌더링과 페이지네이션 보조 로직을 포함한 Vue SFC다. 내부 유틸 함수들이 선언식으로 작성되어 Vue 규칙과 충돌한다.
- `src/app/components/table/Paginator.vue`: 페이지 이동 관련 핸들러를 Vue SFC 내부 명명 함수로 작성하고 있어 스와핑 대상이다.
- `src/app/components/players/PlayerListTable.vue`: 이벤트 핸들러가 `function` / `async function` 형태라 Vue 규칙 위반이다.
- `src/app/composables/query/*.ts`: `execute`, `clear`, `refetch` 등을 `const ... = () => {}` 패턴으로 선언하고 있어 비 Vue 규칙 위반이다.
- `src/app/stores/player.store.ts`: 스토어 액션 다수가 화살표 함수로 선언되어 비 Vue 규칙 위반이다.
- `src/server/utils/auth.ts`: 인증 컨텍스트 생성 및 권한 체크 보조 함수가 화살표 함수 위주로 작성되어 있으며, 후속 스와핑 시 클로저 기반 반환 구조에 가장 큰 영향이 예상된다.
- `src/server/api/characters/*.get.ts`: API 핸들러 내부 합산 보조 함수가 화살표 함수로 작성되어 비 Vue 규칙 위반이다.

## 영향도 평가
### 직접 영향
- Vue SFC에서는 `function`을 `const fn = () => {}`로 바꾸는 수준이라 호출부 영향이 작다.
- 비 Vue 파일에서는 `const fn = () => {}`를 `function fn() {}`로 전환해야 하므로 선언 호이스팅과 `this` 바인딩은 유리해지지만, 타입 파라미터와 오버로드 표현 방식이 달라질 수 있다.
- `src/server/utils/auth.ts`는 반환 객체 내부에서 바깥 스코프 값을 참조하는 함수가 많아, 일괄 치환 시 순서와 명칭 충돌을 주의해야 한다.

### 간접 영향
- 향후 ESLint 규칙이나 코드 생성 규칙으로 같은 정책을 강제하지 않으면 동일 패턴이 재유입될 수 있다.
- 현재 점검은 명명 함수 선언만 대상으로 했으므로, 마스터가 "인라인 콜백까지 전부 선언식/화살표로 통일"을 의도했다면 범위가 추가 확장될 수 있다.
- Nuxt/Vue 컴포저블과 Nitro 핸들러는 함수 표현식 패턴을 자주 사용하므로, 스타일 치환 후 타입 추론이나 선언 위치가 바뀌면서 미세한 포맷 차이가 발생할 수 있다.

### 예상 기술 난제
- 타입 파라미터를 가진 화살표 함수는 함수 선언식으로 옮길 때 제네릭 위치와 반환 타입 표기를 조정해야 한다.
- 스토어나 인증 유틸처럼 같은 스코프에서 상수와 함수가 섞여 있는 파일은 선언 순서 재배치 없이 안전하게 스와핑해야 한다.
- API 핸들러 내부 지역 함수는 로직은 단순하지만, 포맷 변경 후 린트나 타입체크가 추가로 깨지지 않는지 확인이 필요하다.

### 가정 / 불확실성
- 현재 세션에서는 `context7 MCP`를 사용할 수 없어 로컬 검색 결과만 근거로 삼았다.
- 이번 점검은 명명된 함수 선언만 기준으로 집계했다. 인라인 콜백까지 정책 대상에 포함하는지는 아직 확정되지 않았다.
- 사용자 요청의 "app, server 전부"를 `src/app`, `src/server`로 해석했다. 루트 `src/app.vue`와 설정 파일은 이번 범위에서 제외했다.
