# 분석 (Analysis)

## 요약
- 요청 내용: 생성 버튼 라벨을 `캐릭터 생성`으로 바꾸고, 캠페인을 드롭다운 선택 방식으로 제공한다.
- 범위: `CharacterEditorForm.vue`
- 구현 가능성: 가능

## 요구사항 요약
- 생성 모드 버튼 라벨은 UI 확인 문구가 아니라 실제 동작명인 `캐릭터 생성`이어야 한다.
- 캐릭터는 캠페인을 선택할 수 있어야 한다.
- 캠페인 선택 여부에 따라 후속 API 경로가 달라지므로 submit payload에도 캠페인 식별자가 필요하다.

## 현재 상태
### 관련 파일
- `src/app/components/characters/CharacterEditorForm.vue`
  - 현재 간소화된 캐릭터 폼이며, 캠페인 선택 필드가 제거되어 있다.
- `src/app/composables/campaigns/useGetCampaignList.ts`
  - `/campaigns` 목록을 조회해 캠페인 스토어에 적재한다.
- `src/app/stores/campaign.store.ts`
  - `campaignList` 상태를 제공한다.
- `src/server/types/dto/campaign.dto.ts`
  - 드롭다운에 사용할 캠페인 `id`, `name` 필드를 제공한다.

### 현재 로직 분석
- 생성/수정 버튼 라벨은 `생성 UI 확인`, `수정 UI 확인`으로 표시된다.
- submit payload에는 `campaignId`가 없다.
- 캠페인 목록을 읽는 인프라는 이미 존재하므로 폼에서 목록 조회만 연결하면 된다.

## 영향도 평가
### 직접 영향
- `src/app/components/characters/CharacterEditorForm.vue`

### 간접 영향
- 후속 저장 로직은 `campaignId` 존재 여부에 따라 다른 API를 고를 수 있게 된다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- 드롭다운 선택값은 `number | null`로 관리해야 하며, 수정 모드 기본값도 동기화해야 한다.
- 목록 조회 실패 시에도 폼이 깨지지 않도록 빈 옵션 상태를 허용해야 한다.

### 가정 / 불확실성
- 캠페인 목록은 전체 `/campaigns` 기준으로 제공한다.
- 캠페인 미선택 상태는 `null`로 유지한다.
