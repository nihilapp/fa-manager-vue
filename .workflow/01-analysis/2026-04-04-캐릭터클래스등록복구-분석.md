# 분석 (Analysis)

## 요약
- 요청 내용: 캐릭터 클래스 정보가 등록되지 않는 문제의 원인을 찾고 수정
- 범위: 캐릭터 생성 폼의 클래스 전송 로직, `characters/[id]/classes` 서버 API
- 구현 가능성: 가능

## 요구사항 요약
- 캐릭터 생성 시 입력한 클래스 정보가 실제 저장되어야 한다.
- 프론트와 백엔드 중 어느 쪽이 문제인지 판별하고, 실제 장애 지점을 복구해야 한다.

## 현재 상태
### 관련 파일
- `src/app/components/characters/CharacterCreateForm.vue`: 캐릭터 생성 후 클래스 생성 요청을 순차 호출한다.
- `src/app/stores/character.store.ts`: 클래스 입력값을 payload로 정규화한다.
- `src/app/composables/characters/useCreateCharacterClass.ts`: 클래스 생성 API 호출 래퍼다.
- `src/server/api/characters/[id]/classes/index.post.ts`: 클래스 생성 엔드포인트여야 하나 실제로는 캐릭터 목록 조회 코드가 들어 있었다.
- `src/server/api/characters/[id]/classes/[className].put.ts`: 클래스 수정 엔드포인트여야 하나 잘못된 복붙 코드였다.
- `src/server/api/characters/[id]/classes/[className].delete.ts`: 클래스 삭제 엔드포인트여야 하나 잘못된 복붙 코드였다.

### 현재 로직 분석
- 프론트는 캐릭터 생성 성공 후 클래스 payload 배열을 순회하며 `/characters/{id}/classes` POST를 호출한다.
- 하지만 서버의 POST/PUT/DELETE 파일 3개가 모두 동일한 잘못된 목록 조회 코드여서 실제 저장/수정/삭제가 수행되지 않는다.
- 프론트도 `subClassName`이 비어 있으면 payload에서 해당 클래스를 제외하는 필터가 있어 기본 클래스만 입력한 경우 등록이 누락된다.

## 영향도 평가
### 직접 영향
- 캐릭터 클래스 생성/수정/삭제 API
- 캐릭터 생성 폼의 클래스 payload 생성 조건

### 간접 영향
- 캐릭터 상세/편집 화면의 클래스 표시 정합성
- 캐릭터 현재 레벨 계산에 쓰이는 클래스 데이터 반영

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
