# 분석 (Analysis)

## 요약
- 요청 내용: `Box` 공통 컴포넌트에서 `padding` 프롭을 제거하고, `border`와 `color` 프롭만 유지하도록 전면 수정한다
- 범위: `src/app/components/common/Box.vue` 및 현재 `Box` 호출부 전반
- 구현 가능성: 가능

## 요구사항 요약
- `Box`의 디자인 요소는 기본적으로 `class` 프롭으로 전달한다
- 반복 사용 가치가 있는 항목만 공통 프롭으로 유지한다
- 새 프롭 이름은 `border`와 `color`로 정리한다
- `color`는 테두리색과 배경색을 함께 제어하며, 값은 `src/app/types/common.types.ts`의 `StatusColor` 색상 이름만 사용한다
- 색상 매핑은 `StatusColor` 이름에 맞춰 `Box` 내부에서 직접 정의한다

## 현재 상태
### 관련 파일
- `src/app/components/common/Box.vue`
  - 현재 `borderStyle`, `borderColor`, `padding` 프롭을 제공하며, `borderColor`는 `slate`, `stone`만 허용한다
- `src/app/types/common.types.ts`
  - `StatusColor`를 `gray`, `red`, `rose`, `orange`, `yellow`, `green`, `blue`, `purple`, `pink`, `black`으로 정의한다
- `src/app/components/common/StatusBadge.vue`
  - `StatusColor` 기반으로 border/background/text 색을 매핑하는 기준 예시다
- `src/app/components/characters/CharacterCreateForm.vue`
  - `Box`에 `border-style`, `border-color`, `padding`을 사용한다
- `src/app/components/characters/CharacterEditorForm.vue`
  - 로딩/에러 박스와 섹션 박스에서 `padding`과 `border-style`, `border-color`를 사용한다
- `src/app/components/characters/CharacterProfileDetail.vue`
  - 여러 섹션이 `padding` 프롭을 사용한다
- `src/app/components/players/PlayerProfileDetail.vue`
  - 일부 `Box`는 이미 `class` 기반 padding을 사용한다
- `src/app/pages/characters/detail/[id].vue`
  - 빈 상태 메시지 박스에 `border-style`, `border-color`, `padding`을 사용한다

### 현재 로직 분석
- `Box`는 기본으로 `rounded-2 bg-white`와 border/padding variant를 함께 묶고 있다
- 현재 `padding` 프롭 사용처는 많지만, 대부분 `class`에 동일 계열 레이아웃 클래스가 이미 섞여 있어 `p-*`를 `class`로 옮겨도 구조상 문제는 적다
- `border-color="red"` 같은 잘못된 호출이 이미 존재해 현재 계약은 호출부 실수 방지 측면에서도 충분히 직관적이지 않다
- `StatusColor`와 `StatusBadge`가 이미 공통 색상 이름 체계를 제공하므로 `Box`도 이를 재사용하면 일관성이 좋아진다
- context7 MCP 서버는 현재 세션에 노출되지 않아 외부 검색 없이 로컬 코드 기준으로 분석했다

## 영향도 평가
### 직접 영향
- `src/app/components/common/Box.vue`
- `src/app/components/characters/CharacterCreateForm.vue`
- `src/app/components/characters/CharacterEditorForm.vue`
- `src/app/components/characters/CharacterProfileDetail.vue`
- `src/app/pages/characters/detail/[id].vue`

### 간접 영향
- 향후 `Box`를 사용하는 다른 화면은 `padding` 프롭 제거 이후 모두 `class` 중심 사용 패턴을 따르게 된다
- `color` prop이 배경색까지 함께 결정하므로, 기존 배경색을 `class`로 따로 주는 화면은 우선순위 충돌 여부를 확인해야 한다

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- `StatusColor`별 `Box` 색상 톤을 직접 정의해야 하며, `StatusBadge`와 완전히 같게 할지 박스 용도에 맞게 더 연한 톤으로 조정할지 판단이 필요하다
- 기존 `padding` 프롭 호출부를 일괄 `class`로 치환할 때 누락을 방지해야 한다

### 가정 / 불확실성
- 이번 정비에서는 `Box` 기본 배경은 `white`를 유지하되 `color`가 지정되면 해당 색 계열의 border/background를 함께 적용하는 방향이 적절하다
- `text` 색상은 `Box`가 강제하지 않고 호출부 `class`에 맡기는 쪽이 안전하다
