# 분석 (Analysis)

## 요약
- 요청 내용: 완료된 워크플로우는 `history.md`에 최신순으로 기록하고, 진행 중 작업은 `.workflow/TODO.md`로 분리 관리
- 범위: `.workflow` 운영 문서, 템플릿, 이번 작업 절차 문서
- 구현 가능성: 가능

## 요구사항 요약
- `history.md`는 최신 완료 작업이 가장 위에 오도록 정렬해야 한다.
- 현재 진행 중인 작업은 `TODO.md`에서만 관리하고, 작업 완료 시 즉시 `history.md`로 이동시키는 운영 규칙이 필요하다.
- `TODO.md`는 세션 재개 시 먼저 읽는 기준 문서 역할을 해야 한다.
- 이번 작업은 테스트 목적이므로 절차 문서를 생성해야 한다.

## 현재 상태
### 관련 파일
- `.workflow/history.md`: 완료된 작업 목록을 보관하지만 최신순 정렬 규칙이 반영되어 있지 않음
- `.workflow/template/YYYY-MM-DD-작업-태스크확립.md`: 체크리스트 문서 기준은 있으나 전체 진행 상태를 모아보는 중앙 문서는 없음
- `.workflow/template/YYYY-MM-DD-작업-보고.md`: 보고서 파일명 규칙은 갱신되었으나 `history/TODO` 운영 규칙은 없음
- `.workflow/03-tasking/2026-03-22-authHelper사용방법분석-태스크확립.md`: 현재 미완료 작업의 근거 문서

### 현재 로직 분석
- 완료된 작업은 단계별 실파일을 정리한 뒤 `history.md`에만 남기는 방향으로 운영되고 있다.
- 진행 중 작업은 단계 폴더에 흩어져 있어 현재 활성/보류 상태를 한눈에 파악하기 어렵다.
