# 20260320-수동앱임포트타입정비-분석

## 참조 문서
- `.workflow/00-init/20260320-수동앱임포트타입정비-초기화.md`
- `.workflow/06-reports/REPORT-v0.0.4-프론트오토임포트정비.md`

## 1. 요구사항 요약
- 마스터 요청: `src/app/types/manual-app-imports.d.ts`가 실제 import 기반으로 타입을 제공하도록 수정하여 전역 심볼이 `any`로 보이지 않게 정비합니다.
- 구현 해석: `#imports` 단일 집합 타입 인덱싱 대신, 각 심볼이 속한 실제 모듈(`vue`, `pinia`, `nuxt/app` 등)의 공개 타입 시그니처를 직접 참조해야 합니다.

## 2. 현황 파악
- 대상 파일은 `type AppImports = typeof import('#imports')`를 선언한 뒤 대부분의 전역 상수와 타입 별칭을 `AppImports['...']` 형식으로 매핑하고 있습니다.
- Nuxt 생성 파일 `.nuxt/types/imports.d.ts`에는 동일 심볼들이 실제 모듈 시그니처로 풀려 있으나, 수동 선언 파일에서는 이 간접 참조가 편집기/언어 서비스 환경에서 안정적으로 전달되지 않는 정황이 확인되었습니다.
- `vue` 전역 컴포넌트 선언은 `#components` 기반이라 현재 문제의 직접 원인으로 보이지 않았습니다.

## 3. 관련 파일 및 역할
- `src/app/types/manual-app-imports.d.ts`: 프론트 전역 심볼과 공용 타입 별칭을 보완하는 수동 선언 파일입니다.
- `tsconfig.json`: `.nuxt/types/imports.d.ts`, `.nuxt/types/components.d.ts`, `src/**/*`를 포함하여 수동 선언과 생성 선언이 함께 타입 시스템에 진입하도록 구성합니다.
- `.nuxt/types/imports.d.ts`: Nuxt가 생성한 실제 오토임포트 전역 타입 선언 기준입니다.
- `node_modules/nuxt/dist/app/index.d.ts`: `useRoute`, `navigateTo`, `defineNuxtPlugin` 등 공개 엔트리포인트를 제공합니다.
- `node_modules/@pinia/nuxt/dist/runtime/composables.d.ts`: `usePinia` 선언을 제공합니다.

## 4. 영향도 평가
- 직접 영향:
  - `ref`, `computed`, `defineStore`, `useRoute`, `useRuntimeConfig` 등 전역 심볼의 편집기 타입 표시와 추론 결과가 바뀝니다.
  - `UseRoute`, `AppRoute`, `DefineStore` 등 전역 타입 별칭이 실제 시그니처를 직접 반영합니다.
- 간접 영향:
  - 선언 충돌이 발생하면 프론트 전역 타입 전반에 오류가 전파될 수 있으므로 타입체크 확인이 필요합니다.
  - Nuxt/Pnia 버전 업데이트 시 공개 엔트리포인트가 유지되는 한 현재 선언은 기존보다 덜 취약합니다.

## 5. 불확실성 및 판단
- `usePinia`는 `pinia` 본체가 아니라 `@pinia/nuxt` 런타임 컴포저블에서 선언되므로 별도 경로를 사용해야 합니다.
- `definePageMeta` 등 Nuxt 관련 함수는 `nuxt/app` 공개 엔트리포인트에서 확인되어 해당 경로 사용이 가능하다고 판단하였습니다.
- 따라서 설계 단계에서는 `#imports` 의존을 제거하고 실제 모듈별 타입 맵으로 선언을 재구성합니다.
