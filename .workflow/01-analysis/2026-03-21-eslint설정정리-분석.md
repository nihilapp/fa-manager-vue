# 분석 (Analysis)

## 요약
- 요청 내용: `eslint.config.mjs`에서 `.vue` 내부의 TypeScript가 일반 `.ts` 파일과 동일한 TypeScript 규칙을 따르도록 정리
- 범위: ESLint 플랫 설정의 파일 매칭, 파서 연결, TypeScript 공통 규칙 정의, Vue 전용 규칙 분리 방식
- 구현 가능성: 가능

## 요구사항 요약
- `.vue` SFC 내부의 `<script setup lang="ts">`가 일반 `.ts` 파일과 동일한 TypeScript 규칙 세트를 따라야 한다.
- Vue 템플릿 전용 규칙은 유지하되, TypeScript 규칙은 중복 정의 없이 한 기준에서 관리되어야 한다.
- 설정 병합이 필요하더라도 `.ts` 파일 전체를 Vue 파서로 처리하는 식의 과도한 결합은 피해야 한다.

## 현재 상태
### 관련 파일
- `eslint.config.mjs`: 프로젝트 전체 ESLint 플랫 설정 파일. JS/TS 공통 규칙, TS 파서 설정, Vue 파서 설정, import-x 및 vue 접근성 규칙을 모두 포함한다.
- `package.json`: ESLint, `typescript-eslint`, `vue-eslint-parser`, `eslint-plugin-vue` 등 설정 구성에 사용되는 패키지 버전을 정의한다.
- `PRD/Coding-Rules.md`: 프로젝트 전반의 구현 규칙을 제공하지만 ESLint 세부 구조 자체는 아직 문서화되어 있지 않다.

### 현재 로직 분석
- `eslint.config.mjs` 초반부에서 `js.configs.recommended`, `...tseslint.configs.recommended`, `@stylistic` 설정을 전역으로 로드한다.
- `files: ['**/*.{js,ts}']` 블록에서 공통 규칙과 TypeScript 관련 규칙 일부를 정의한다.
- `files: ['**/*.ts']` 블록에서 `.ts` 파일만 `tseslint.parser`를 직접 사용하도록 지정한다.
- `files: ['**/*.{js,mjs,ts,vue}', 'eslint.config.{js,mjs,ts}']` 블록에서 `vue-eslint-parser`를 사용하고 `parserOptions.parser = tsParser`를 지정해 Vue SFC와 일부 JS/TS 파일까지 함께 처리한다.
- 같은 파일에서 `@typescript-eslint/no-unused-vars`, `@typescript-eslint/no-shadow`, `@typescript-eslint/no-use-before-define`가 두 군데에 정의되어 있으며, Vue 포함 블록에서 경고 수준으로 다시 선언된다.

## 영향도 평가
### 직접 영향
- `eslint.config.mjs`의 파일 매칭 범위 재조정이 필요하다.
- TypeScript 공통 규칙을 별도 상수 또는 재사용 가능한 규칙 묶음으로 분리해야 한다.
- `.vue` 대상 블록은 Vue 전용 파서와 템플릿 규칙을 유지하되, TypeScript 공통 규칙을 공유하도록 구조 변경이 필요하다.

### 간접 영향
- ESLint 결과가 바뀌면서 기존 `.vue` 파일에서 새 경고 또는 에러가 노출될 수 있다.
- `.ts` 파일에 Vue 파서가 잘못 적용되어 있던 상태를 정리하면, 파일별 파서 책임이 명확해져 추후 규칙 유지보수가 쉬워진다.
- import 관련 규칙과 전역 globals 정의는 기존 동작을 유지해야 하므로 병합 범위를 과도하게 넓히면 부작용이 생길 수 있다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- ESLint 플랫 설정은 배열 순서와 `files` 매칭 범위에 따라 최종 규칙이 달라지므로, 단순 병합보다 블록 책임 분리가 중요하다.
- Vue SFC 내부 TS 규칙 공유를 위해서는 `vue-eslint-parser` 아래의 `parserOptions.parser`와 TypeScript 공통 규칙 재사용을 함께 설계해야 한다.
- 현재 Vue 포함 블록이 `**/*.{js,mjs,ts,vue}`까지 포괄하고 있어, 의도치 않게 `.ts` 파일도 Vue 파서 경로에 들어갈 수 있는 구조인지 주의 깊게 다뤄야 한다.

### 가정 / 불확실성
- 현재 프로젝트에 ESLint 실행 스크립트가 `package.json`에 명시되어 있지 않아 실제 검증 명령은 추후 확정이 필요하다.
- 지침서에서 요구한 context7 MCP 검색은 현재 세션 도구에 노출되어 있지 않아 로컬 코드베이스 기준으로 분석했다.
