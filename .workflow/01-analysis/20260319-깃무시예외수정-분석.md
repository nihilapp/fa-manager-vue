## 요구사항 재해석
- 전역 `logs` 무시 규칙은 유지합니다.
- 단, `src/server/api/sessions/[id]/logs` 경로는 Git 추적 대상에서 제외되지 않도록 예외 처리해야 합니다.
- 예외 처리는 실제 `.gitignore` 문법에 맞아야 하며, 디렉터리와 내부 파일까지 포함되어야 합니다.

## 현황 파악
- 현재 `.gitignore`에는 `logs`, `*.log`, `!src/server/api/sessions/[id]/logs/` 규칙이 있습니다.
- `logs` 규칙 때문에 `src/server/api/sessions/[id]/logs`도 무시됩니다.
- `[id]`는 `.gitignore`에서 문자 클래스 패턴으로 해석되므로, 리터럴 디렉터리명으로 동작하지 않습니다.

## 영향 파일 및 역할
- `.gitignore`: Git 추적 제외/예외 규칙을 정의하는 파일이며, 이번 작업의 직접 수정 대상입니다.
- `src/server/api/sessions/[id]/logs`: 세션 API 하위의 로그 디렉터리 경로이며, 예외 처리의 대상입니다.

## 영향도 평가
- 직접 영향: 해당 경로와 내부 파일이 Git 추적 대상으로 복구됩니다.
- 간접 영향: 다른 `logs` 디렉터리 무시 규칙에는 영향이 없습니다.

## 불확실성
- 해당 경로에 현재 실제 파일이 존재하지 않을 수 있습니다.
- 디렉터리만 존재하고 파일이 없다면 Git은 폴더 자체를 추적하지 않습니다.

## 실행 판단
- `.gitignore`에서 `[id]`를 리터럴로 해석되도록 보정하고, 내부 파일까지 예외 처리하는 규칙을 추가하는 것이 적절합니다.
