# 분석 (Analysis)

## 요약
- 요청 내용: 필요 없는 패키지들을 제거하고, 특히 Vue Query 관련 패키지와 잔존 파일을 정리한다.
- 범위: `package.json`, `pnpm-lock.yaml`, `PRD/Task-List.md`
- 구현 가능성: 가능

## 요구사항 요약
- 앱 코드에서 이미 제거한 Vue Query 관련 파일/설정에 맞춰 패키지 의존성도 제거한다.
- 더 이상 사용하지 않는 `axios`도 함께 정리한다.
- 프로젝트 기준 문서에 남아 있는 "Vue Query 전환" 표현을 현재 방향에 맞게 갱신한다.

## 현재 상태
### 관련 파일
- `package.json`: `@tanstack/vue-query`, `@tanstack/vue-query-devtools`, `axios`가 남아 있음
- `pnpm-lock.yaml`: 위 패키지와 전이 의존성이 남아 있음
- `PRD/Task-List.md`: "Vue Query 전환"과 설치 항목이 아직 남아 있음

### 현재 로직 분석
- 앱 코드와 Nuxt 설정에서는 Vue Query/axios 참조가 제거된 상태다.
- 실제 잔존물은 패키지 선언과 PRD 문서 표현에 한정된다.
- `pnpm remove`는 lockfile 갱신을 위해 네트워크 접근이 필요하며, 샌드박스에서는 실패한다.

## 영향도 평가
### 직접 영향
- 패키지 의존성 제거
- lockfile 경량화
- PRD 작업 목록 표현 수정

### 간접 영향
- 개발 환경에서 설치된 `node_modules`가 함께 재정렬될 수 있다.
- lockfile 갱신 과정에서 다른 패키지 버전이 일부 최신 고정값으로 정리될 수 있다.

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- lockfile 갱신은 권한 상승이 필요하다.
- `pnpm remove`가 부수적으로 다른 버전 메타데이터를 재정렬할 수 있으므로 diff 확인이 필요하다.

### 가정 / 불확실성
- `@lukemorales/query-key-factory`는 현재도 사용 중이므로 제거 대상이 아니다.
- `context7 MCP`는 현재 세션에서 사용할 수 없어 로컬 코드 기준으로 분석했다.
