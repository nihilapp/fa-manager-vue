# 분석 (Analysis)

## 요약
- 요청 내용: 오토임포트 또는 전역 선언으로 해결되는 import를 전수 점검하고 불필요한 import를 제거
- 범위: `src/app`, `src/server/types`, `src/server/db/table`의 오토임포트/전역 선언 중복 import
- 구현 가능성: 가능

## 요구사항 요약
- Nuxt 자동 import, `manual-app-imports.d.ts`, `manual-server-imports.d.ts` 기준으로 직접 import가 필요 없는 심볼을 제거한다.
- 오토임포트 대상이 아닌 라이브러리나 실제 의존 import는 유지한다.
- 프로젝트 전역에서 규칙을 일관되게 적용하되, 이번 변경은 안전하게 판정 가능한 파일만 수정한다.

## 현재 상태
### 관련 파일
- `nuxt.config.ts`: 앱 컴포넌트 자동 등록, `imports.dirs`, preset auto import 범위를 정의한다.
- `src/app/types/manual-app-imports.d.ts`: Vue/Nuxt/Pinia 전역 심볼과 주요 앱 타입/유틸 전역 선언을 제공한다.
- `src/server/types/manual-server-imports.d.ts`: H3, Drizzle, DTO 타입 전역 선언을 제공한다.
- `src/app/composables/useSetMeta.ts`: 전역으로 노출된 `OpenGraphType`, `SiteMetadata`를 다시 import하고 있다.
- `src/server/types/dto/*.ts`: DTO 타입 선언 파일 일부가 이미 전역 타입인 DTO/공통 타입을 다시 import하고 있다.
- `src/server/db/table/*.ts`: Nitro 자동임포트 대상인 Drizzle 함수, enum, 테이블, 공통 컬럼을 직접 import하고 있다.
- `.nuxt/imports.d.ts`: Nuxt가 실제로 생성한 auto import/auto type export 결과를 확인할 수 있다.

### 현재 로직 분석
- 앱 영역에서는 `.nuxt/imports.d.ts`에 `OpenGraphType`, `SiteMetadata`, `useSetMeta`, `queryKeys`, Pinia/Vue/Nuxt 심볼이 이미 노출된다.
- 서버 영역에서는 `manual-server-imports.d.ts`가 `CommonInDto`, `UserOutDto`, `CampaignOutDto`, `SessionLogOutDto` 등 DTO 계층 타입을 전역으로 선언한다.
- 따라서 DTO 선언 파일 내부의 일부 `import type`는 타입 해석에 필요하지 않고 중복이다.
- 테이블 선언 파일 내부의 `pgTable`, `bigint`, `varchar`, `commonColumns`, `usersTable` 등도 Nitro 전역으로 해석 가능하므로 직접 import가 중복이다.
- 반면 `@tanstack/vue-query`의 `UseMutationReturnType`, `UseQueryReturnType` 같은 원시 라이브러리 타입은 현재 전역 auto import 대상이 아니므로 유지해야 한다.

## 영향도 평가
### 직접 영향
- `src/app/composables/useSetMeta.ts`의 타입 import 제거
- `src/server/types/dto/user.dto.ts` 등 DTO 선언 파일의 중복 type import 제거
- `src/server/db/table/*.ts`의 중복 값 import 제거

### 간접 영향
- TypeScript 전역 선언 해석이 실제로 정상 동작하는지 확인 필요
- DTO 선언 간 순환 참조가 import 제거 후에도 타입 수준에서 유지되는지 확인 필요
- 기존 워크트리 변경이 많아 unrelated 파일은 절대 건드리지 않아야 함

## 실행 판단
### 구현 가능성
- 가능

### 예상 기술 난제
- `manual-server-imports.d.ts`와 DTO 선언 파일 간 순환 타입 관계가 있어 제거 범위를 과도하게 넓히면 안정성을 해칠 수 있음
- 자동 import는 값과 타입의 범위가 다르므로 `.nuxt/imports.d.ts`와 수동 선언 파일을 모두 대조해야 함

### 가정 / 불확실성
- `context7 MCP`는 현재 세션에서 사용할 수 없어 로컬 설정과 생성 파일 기준으로 판정함
- 이번 작업은 명백히 안전한 중복 import만 제거하고, 경계가 애매한 import는 유지함
