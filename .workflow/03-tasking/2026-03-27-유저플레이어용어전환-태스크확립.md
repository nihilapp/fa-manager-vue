## 작업 목록
- [x] Task 1: 프런트 플레이어 도메인 명칭 전환
목표: 스토어, 컴포저블, 컴포넌트, 쿼리 키, 페이지 참조를 `player` 기준으로 정렬한다.
완료 조건: `usePlayerStore`, `PlayerListTable`, `useGetPlayerList`, `queryKeys.players` 기준으로 프런트 호출이 동작 가능한 상태가 된다.
의존성: 없음

- [x] Task 2: 서버 플레이어 API 경로 및 핸들러 전환
목표: `src/server/api/users/*`를 `src/server/api/players/*` 기준으로 전환한다.
완료 조건: 플레이어 CRUD 및 `me`, `check` 엔드포인트가 `/api/players` 경로를 기준으로 선언된다.
의존성: Task 1

- [x] Task 3: 서버 DTO 및 자동임포트 타입 전환
목표: `User*` DTO와 전역 타입을 `Player*` 기준으로 정렬한다.
완료 조건: 서버 DTO 참조와 자동임포트 선언에서 `Player*`가 기본 타입으로 사용된다.
의존성: Task 2

- [x] Task 4: DB 테이블/관계 명칭 전환
목표: `usersTable`과 관련 relation/Drizzle export를 `playersTable` 기준으로 전환한다.
완료 조건: 서버 코드가 `playersTable`을 기준으로 빌드 가능한 구조가 된다.
의존성: Task 3

- [x] Task 5: PRD 및 프로젝트 문서 정리
목표: API 명세, DDL, Task List의 `users` 용어를 `players` 기준으로 갱신한다.
완료 조건: 현재 기준 문서가 새 플레이어 명칭 체계를 반영한다.
의존성: Task 4

- [x] Task 6: 검증 및 마감 문서 작성
목표: 설계 대비 구현 결과를 검증하고 검증/보고 문서를 작성한다.
완료 조건: 검증 문서, 최종 보고서, TODO/history 상태가 모두 정리된다.
의존성: Task 5
