## 작업 목록
- [x] Task 1: 헤더 사용자 정보 null-safe 렌더링 적용
목표: `src/app/components/layout/common-layout/CommonHeader.vue`에서 `myInfo`가 `null`이어도 SSR 예외 없이 렌더되도록 사용자 표시 영역을 방어적으로 수정한다.
완료 조건: `myInfo!.name` 직접 접근이 제거되고, 사용자 정보가 없는 초기 상태에서도 헤더 템플릿이 안전하게 렌더된다.
의존성: `.workflow/02-design/2026-03-23-헤더사용자정보SSR오류분석-설계.md`

- [x] Task 2: 사용자 정보 조회 초기화 시점 조정
목표: `src/app/app.vue`와 `src/app/stores/user.store.ts`를 수정해 `getMyInfo()`가 `onMounted` 전용 흐름이 아니라 앱 초기 진입 시점에서 1회 실행되도록 정리한다.
완료 조건: `/block` 예외를 유지하면서도 사용자 정보 조회가 SSR/CSR 공통 초기화 흐름에서 호출되고, 실패 시 SSR 예외 없이 null-safe 상태를 유지한다.
의존성: Task 1

- [x] Task 3: 관리자 메뉴 반응성 정리 및 최소 검증
목표: `src/app/stores/common.store.ts`의 관리자 메뉴 공개 여부가 `isAdmin` 변화에 따라 반영되도록 조정하고, 헤더 SSR 오류 재발 여부를 코드 기준으로 검증한다.
완료 조건: 관리자 메뉴 공개 조건이 정적 스냅샷이 아닌 반응형 계산으로 정리되고, 구현 결과 및 검증 메모가 후속 검증 단계로 넘길 수 있을 수준으로 정리된다.
의존성: Task 2
