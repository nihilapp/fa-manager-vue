## 작업 목록
- [ ] Task 1: player/session 스토어 액션과 상태 구조 표준화
  - 목표: 기본 축과 플레이어별 파생 축을 구분하고 목록/개별/페이지 액션 패턴을 통일한다.
  - 완료 조건: `player.store.ts`, `session.store.ts`에 표준 액션이 정리되고 기존 호출부와 호환된다.
  - 의존성: 없음

- [ ] Task 2: 기존 players 컴포저블을 스토어 액션 중심으로 재정리
  - 목표: 조회/변경 성공 및 실패 시 저장/초기화 규칙을 통일한다.
  - 완료 조건: 기존 6개 players 컴포저블이 정리되고 스토어 저장 흐름이 일관된다.
  - 의존성: Task 1

- [ ] Task 3: 미구현 players API 대응 컴포저블 추가
  - 목표: `/players/check`, `/players/:id/campaigns`, `/players/me` 수정/삭제, `/players/:id` 삭제 대응을 추가한다.
  - 완료 조건: 신규 컴포저블이 추가되고 query key 및 스토어 저장 패턴과 연결된다.
  - 의존성: Task 1, Task 2

- [ ] Task 4: 관련 화면 참조와 최소 검증 정리
  - 목표: 플레이어 목록/상세/헤더 등 기존 사용처가 새 스토어 구조와 호환되는지 확인한다.
  - 완료 조건: 필요한 참조 수정과 최소 실행 검증 결과가 구현 메모에 반영된다.
  - 의존성: Task 2, Task 3
