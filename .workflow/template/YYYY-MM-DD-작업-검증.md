# 5단계: 검증 (Verification - Check)

## 목적

모든 체크리스트 완료 이후, 실제 결과물이 설계 문서와 체크리스트를 충실히 이행했는지 최종적으로 대조하는 단계이다. 단순 테스트 통과 여부만 보는 것이 아니라, 처음 계획한 구조와 구현 결과가 일치하는지를 검토해야 한다.

## 파일명 규칙

`YYYY-MM-DD-작업명-검증.md`

## 단계 지침

모든 체크리스트가 완료된 후, 2단계 설계와 3단계 절차가 누락 없이 이행되었는지 최종 대조하십시오.

## 입력

* Design 문서
* Checklist 문서
* 구현된 코드
* 테스트 결과

## 반드시 수행할 작업

1. 설계 항목별로 실제 구현이 일치하는지 대조한다.
2. 체크리스트의 모든 작업이 누락 없이 반영되었는지 확인한다.
3. 단위 테스트, 수동 테스트, 통합 테스트 결과를 정리한다.
4. 미결 사항과 사이드 이펙트를 명시한다.
5. 완료 선언 가능 여부를 판단한다.

## 작성 내용

### 1. 설계 준수 여부

2단계 설계 문서에 적힌 내용과 실제 코드가 일치하는지 항목별로 비교한다. 다르다면 왜 달라졌는지, 정당한 변경인지, 문서가 누락되었는지를 적는다.

### 2. 테스트 결과

단위 테스트, 수동 테스트, 통합 테스트 등의 통과 여부를 기록한다. 테스트를 수행하지 못했다면 그 이유와 위험도를 적어야 한다.

### 3. 미결 사항

해결하지 못한 문제, 미룬 작업, 잠재적 사이드 이펙트, 추가 확인이 필요한 항목을 적는다.

### 단위 테스트
-

### 수동 테스트
-

## 완료 조건

* 설계와 실제 구현의 일치 여부가 검토되어 있어야 한다.
* 체크리스트 전 항목의 이행 여부가 확인되어 있어야 한다.
* 테스트 결과가 정리되어 있어야 한다.
* 미결 사항이 있으면 숨기지 않고 문서화되어 있어야 한다.
