# 0단계: 초기화 (Phase 0 - Project Init & PRD Baseline)

## 목적

작업을 시작하기 전에 프로젝트의 기준 문서와 폴더 구조가 갖춰져 있는지 확인하고, 부족한 경우 표준 PRD 자산을 생성하거나 기존 문서를 갱신하는 단계이다. 이 단계의 핵심은 이후 분석, 설계, 구현이 흔들리지 않도록 프로젝트의 요구사항 기준점과 작업 관리 기준점을 먼저 고정하는 데 있다.

## 파일명 규칙

`YYYY-MM-DD-작업명-초기화.md`

## 단계 지침

프로젝트 루트의 `PRD/` 및 `.workflow/` 구조를 점검하고, 현재 상태가 `초기화(Initialize)`인지 `갱신(Update)`인지 판단하십시오. 초기화가 필요한 경우 표준 PRD 3종 세트와 작업 문서 구조를 마련해야 합니다.

## 입력

* 사용자 요청 사항
* 프로젝트 루트 폴더 구조
* 기존 `PRD/` 문서
* 기존 `.workflow/` 문서
* 프로젝트 전용 규칙 문서

## 반드시 수행할 작업

1. 루트에 `PRD/` 폴더 존재 여부를 확인한다.
2. `PRD/` 내부의 핵심 문서 존재 여부를 확인한다.
3. `.workflow/` 작업 기록 폴더 구조가 갖춰져 있는지 확인한다.
4. 현재 작업이 `초기화`인지 `갱신`인지 판단한다.
5. 부족한 문서와 폴더를 어떤 기준으로 생성 또는 갱신할지 결정한다.

## 작성 내용

### 1. 현재 상태 점검

`PRD/`와 `.workflow/`의 존재 여부와 현재 상태를 기록한다. 단순히 있다/없다로 끝내지 말고 어떤 문서가 있고 무엇이 비어 있는지까지 적는다.

### 2. 실행 모드 판단

아래 기준으로 이번 작업의 모드를 결정한다.

* `초기화(Initialize)`: `PRD/`가 없거나, PRD 핵심 문서가 비어 있어 기준선 역할을 하지 못하는 경우
* `갱신(Update)`: `PRD/`가 존재하며 기존 기준 문서를 확장 또는 최신화하면 되는 경우

### 3. 생성 또는 갱신 대상

필요한 생성/갱신 대상을 목록화한다.

* `PRD/PRD.md`
* `PRD/Task-List.md`
* `PRD/Coding-Rules.md`
* `.workflow/00-init/`
* `.workflow/01-analysis/`
* `.workflow/02-design/`
* `.workflow/03-tasking/`
* `.workflow/04-implementation/`
* `.workflow/05-verification/`
* `.workflow/06-reports/`

프로젝트 특성상 파일명이 다르다면 대응 관계를 함께 적는다.

### 4. 기준선 확정

이후 1단계부터 어떤 문서를 절대 기준점으로 삼을지 명시한다. 예를 들어, 요구사항 기준은 `PRD.md`, 실행 우선순위 기준은 `Task-List.md`, 구현 규칙 기준은 `Coding-Rules.md`로 확정한다.

## 완료 조건

* 이번 작업이 초기화인지 갱신인지 명확히 판단되어 있어야 한다.
* PRD 핵심 문서 3종과 기록 폴더의 준비 상태가 문서화되어 있어야 한다.
* 이후 분석 단계가 참조할 기준 문서가 확정되어 있어야 한다.
* 누락된 초기 자산이 있으면 무엇을 생성해야 하는지 명확히 정리되어 있어야 한다.
