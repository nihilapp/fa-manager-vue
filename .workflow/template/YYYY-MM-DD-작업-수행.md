# 4단계: 수행 (Implementation - Do)

## 목적

체크리스트에서 현재 활성화된 단 하나의 작업에 집중하여 실제 코드 수정과 구현을 수행하는 단계이다. 이 단계는 문서 작성보다 실행이 중심이며, 설계를 현실의 코드에 반영하는 작업이다.

## 파일명 규칙

`YYYY-MM-DD-작업명-수행.md`

## 단계 지침

3단계의 체크리스트 중 활성화된 항목 하나에만 집중하여 코드를 작성하거나 수정하십시오.

## 수행 방식

별도의 상세 문서 기록보다는 실제 코드 수정을 진행한다. 필요 시 수정 사항에 대한 짧은 메모, 캡처, 비교 스냅샷 등을 `04-implementation` 폴더에 임시 보관할 수 있다.

## 입력

* 활성화된 Checklist 항목
* Design 문서
* 관련 코드 파일

## 반드시 수행할 작업

1. 현재 활성 태스크 하나만 선택한다.
2. 설계에 정의된 방향을 벗어나지 않고 코드를 수정한다.
3. 필요 시 단위 테스트 또는 최소 검증 코드를 함께 작성한다.
4. 구현 도중 설계 변경 필요성이 발생하면 즉시 설계 단계로 되돌아간다.
5. 작업 완료 후 체크리스트를 갱신한다.

## 구현 규칙

* 여러 태스크를 한 번에 병행하지 않는다.
* 설계에 없는 구조 변경을 임의로 수행하지 않는다.
* 리팩토링이 필요한 경우에도 현재 태스크와 직접 관련된 범위 안에서만 수행한다.
* 커밋은 작고 의미 있게 유지한다.

## 권장 메모 템플릿

```md
# 구현 메모 (Implementation Memo)
- 활성 태스크: Task 2
- 수정 파일:
  - `src/...`
  - `src/...`
- 간단 메모:
  - 서비스 레이어에 권한 검증 추가
  - DTO 매핑 수정
```

## 완료 조건

* 현재 활성 태스크의 코드 수정이 끝나 있어야 한다.
* 관련 최소 검증이 수행되어야 한다.
* 체크리스트에 완료 여부가 반영되어야 한다.
* 추가 구조 변경이 필요하면 설계 문서가 먼저 갱신되어야 한다.
