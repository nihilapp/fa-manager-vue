# 6단계: 보고 및 종료 (Final Report - Act)

## 목적

이번 작업의 시작부터 끝까지를 정리하여, 미래의 마스터 혹은 다른 에이전트가 다음 작업을 빠르게 이어갈 수 있도록 기록하는 단계이다. 이 문서는 단순 결과 요약이 아니라, 작업의 맥락과 의사결정, 특이사항을 압축한 유지보수용 인계 문서로 기능해야 한다.

## 파일명 규칙

`REPORT-vX.X.X-작업명.md`

## 단계 지침

이번 작업의 시작부터 끝까지를 정리하여 미래의 마스터(혹은 다른 에이전트)가 참고할 수 있게 기록하십시오.

## 입력

* Analysis 문서
* Design 문서
* Checklist 문서
* Verification 문서
* 실제 코드 변경 내역

## 반드시 수행할 작업

1. 어떤 문제를 해결했는지 작업 개요를 적는다.
2. 주요 수정 파일과 핵심 기능을 요약한다.
3. 작업 중 발견된 특이사항이나 후속 이슈를 기록한다.
4. 이번 작업의 성공 여부를 판단한다.
5. 필요 시 프로젝트 버전 업데이트를 제안한다.

## 작성 내용

### 1. 작업 개요

무엇을 해결하기 위한 작업이었는지, 어떤 요구사항에서 시작되었는지 설명한다.

### 2. 결과 요약

수정된 주요 파일, 추가된 기능, 변경된 흐름, 개선된 부분을 요약한다.

### 3. 특이 사항

예상과 달랐던 점, 새롭게 발견된 기술 부채, 후속 개선이 필요한 부분을 기록한다.

### 4. 결론

작업 성공 여부를 명시하고, 배포 가능 여부나 버전 업데이트 필요성을 제안한다.

## 완료 조건

* 작업의 전 과정이 요약되어 있어야 한다.
* 결과물과 특이사항이 충분히 기록되어 있어야 기재되어야 한다.
* 후속 작업자가 바로 이어받을 수 있을 정도로 맥락이 보존되어 있어야 한다.
* 프로젝트 상태 판단과 다음 액션 제안이 포함되어 있어야 한다.
