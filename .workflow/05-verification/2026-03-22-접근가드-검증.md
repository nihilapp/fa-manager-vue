## 테스트 결과
### 단위 테스트
- 별도 단위 테스트는 작성하지 않았다.

### 수동 테스트
- 코드 경로 기준 수동 점검:
  - `X-Discord-ID`가 없으면 `checkCommunityAccess`가 `UNAUTHORIZED`를 반환하고 서버 미들웨어가 `/block`으로 리다이렉트한다.
  - 개발 환경이 아니면 `TEAM_ONLY`를 반환한다.
  - 존재하지 않는 `discordId`면 `NOT_FOUND`를 반환한다.
  - `/block` 경로는 예외 처리되어 리다이렉트 루프가 생기지 않는다.

### 통합 테스트
- `pnpm.cmd exec eslint src/server/utils/access.ts src/server/middleware/access.ts src/server/api/users/check.get.ts src/app/pages/block.vue src/app/components/layout/common-layout/CommonHeader.vue src/app/layouts/main.vue src/app/utils/api-client.ts src/app/stores/user.store.ts` 실행 성공
- `pnpm.cmd exec nuxi typecheck`는 `vue-tsc` 패키지 조회 시 네트워크/권한 문제로 실패하여 수행 완료하지 못했다.

## 미결 사항
- 이 구조는 상위 프록시나 진입 경로에서 실제로 `X-Discord-ID` 요청 헤더를 페이지 요청에 붙여 준다는 전제를 가진다.
- 브라우저 단독 환경만으로는 근본적인 위조 방지가 불가능하므로, 운영 확장 시에는 로그인 또는 서명된 게이트웨이 토큰 구조가 필요하다.
- 클라이언트 사이드 내비게이션 자체를 추가 검증하지는 않지만, 직접 진입/새로고침/검색 유입 차단 목적에는 서버 미들웨어가 더 적합하다.
