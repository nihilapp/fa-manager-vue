## 설계 대비 구현 대조
- 설계 항목: 앱 공용 타입과 경로 기준 정리
  - 구현 결과: 앱 전용 응답 타입 별칭을 제거하고 서버 전역 타입 `BaseResponse`, `ListData`를 프론트에서 직접 사용하도록 정리했습니다.
- 설계 항목: 쿼리 컴포저블 공통화
  - 구현 결과: `src/app/composables/query/useApiRequest.ts`를 추가하고 `usePost`, `usePut`, `usePatch`, `useDelete`를 공통 mutation 헬퍼 기반으로 재구성했습니다.
- 설계 항목: `useGet` 실행 제어 정리
  - 구현 결과: `body` 인자를 제거하고 `enabled`, `query`, `key` 옵션 중심으로 정리했으며 중복 실행 가능성을 제거했습니다.
- 설계 항목: `useSetMeta` 메타 해석 일관화
  - 구현 결과: `useSeoMeta`와 `useHead`를 병행 사용하고 제목, canonical, 이미지 URL 조합 규칙을 정리했습니다.
- 설계 항목: 앱 선언형 오토임포트 적용
  - 구현 결과: `nuxt.config.ts`에 앱 `imports`의 `dirs`, `imports`, `presets`를 추가했고 `manual-app-imports.d.ts`는 충돌 방지를 위해 최소 파일로 정리했습니다.
- 설계 항목: 백엔드 기준 타입명 통일
  - 구현 결과: `src/server/types/manual-server-imports.d.ts`의 전역 `BaseResponse`, `ListData`를 실제 응답 형태 기준으로 보정했고 프론트 사용처에서 해당 이름을 사용하도록 변경했습니다.

## 테스트 결과
- `pnpm.cmd exec tsc --noEmit`
  - 결과: 통과
- `pnpm.cmd exec nuxt typecheck`
  - 결과: 실패
  - 사유: `vue-tsc`가 로컬 설치되어 있지 않아 외부 레지스트리 다운로드가 필요했으나 현재 세션의 권한/네트워크 제약으로 실행되지 않았습니다.

## 미결 사항
- `nuxt typecheck`를 완전히 통과시키려면 `vue-tsc`를 로컬에 설치하거나 네트워크가 허용된 환경에서 재검증해야 합니다.
- `appConfig.site.url`의 운영값이 비어 있어 실제 배포 전에는 canonical/OG 절대 URL 기준값을 별도 설정해야 합니다.
