## 설계 준수 여부
- 항목 1: CRUD composable의 공용 export 표면 축소
  - 설계 기대값: `useDelete.ts`, `usePatch.ts`, `usePost.ts`, `usePut.ts`에서 함수 export만 남고 내부 타입은 파일 내부에 머문다.
  - 실제 구현: 네 파일 모두 `UseXxxReturn`, `UseXxxOptions`를 비공개 `interface`로 유지하고, 최종 export는 `useXxx`만 남겼다.
  - 일치 여부: 일치
- 항목 2: exported generic function 시그니처 분리
  - 설계 기대값: 오토임포트 스캐너가 `TBody`를 top-level export로 인식하지 않도록 exported 구문에서 제네릭을 제거한다.
  - 실제 구현: 네 파일 모두 `createUseXxx` 내부 제네릭 함수와 `export const useXxx = createUseXxx` 구조로 변경했다.
  - 일치 여부: 일치
- 항목 3: `.nuxt` 재생성으로 결과 확인
  - 설계 기대값: `.nuxt/imports.d.ts`, `.nuxt/types/imports.d.ts`에 `TBody`가 더 이상 생성되지 않는다.
  - 실제 구현: `pnpm.cmd run postinstall` 실행 후 `rg -n "\\bTBody\\b" .nuxt\\imports.d.ts .nuxt\\types\\imports.d.ts` 결과가 비어 있었다.
  - 일치 여부: 일치
