# 20260320-뷰쿼리전환-검증

## 1. 검증 개요
기본 데이터 패칭 구조(`useFetch`, `$fetch`)에서 `@tanstack/vue-query`로의 전환 및 쿼리 키 빌더 도입 결과에 대한 최종 검증을 수행합니다.

## 2. 체크리스트 대조 결과
- [x] **패키지 설치**: `@tanstack/vue-query`, `@lukemorales/query-key-factory` 설치 완료.
- [x] **SSR 하이드레이션**: `src/app/plugins/vue-query.ts`를 통해 SSR 시 데이터 직렬화 및 클라이언트 복원 설정 완료.
- [x] **쿼리 키 팩토리**: `src/app/utils/query-keys.ts`에 도메인별(Users, Campaigns, Characters) 쿼리 키 정의 및 `mergeQueryKeys` 적용 완료.
- [x] **커스텀 컴포저블 리팩토링**:
  - `useGet`: `useQuery` 및 `suspense()` 기반으로 재구현하여 `await` 호출 호환성 유지.
  - `useMutation` 기반: `createApiMutation` 리팩토링을 통해 모든 뮤테이션 컴포저블(`usePost` 등) 개선.
- [x] **하위 호환성**: 기존 `user.store.ts`에서 호출 방식 변경 없이 동작 확인 (타입 체크 완료).

## 3. 주요 개선 사항
- **타입 안정성**: `query-key-factory`를 통해 캐시 키 관리 시 발생할 수 있는 오타 방지 및 타입 추론 지원.
- **캐싱 최적화**: `Vue Query`의 기본 `staleTime` (5분) 설정을 통해 불필요한 API 재호청 감소.
- **개발 도구**: `VueQueryDevtools`를 추가하여 브라우저에서 실시간 쿼리 모니터링 가능.

## 4. 미결 사항 및 후속 작업
- **캐시 무효화**: 현재 뮤테이션 성공 시 자동 캐시 무효화 로직이 모든 지점에 적용된 것은 아니므로, 개별 도메인 로직 구현 시 `queryClient.invalidateQueries` 호출 필요.
- **Suspense 활용**: 향후 복잡한 페이지 로딩 시 Nuxt `<Suspense>`와 더욱 긴밀하게 연동할 수 있는 구조 확보.

## 5. 최종 판정
**배포 가능**. 모든 코드가 타입 안전하며 기존 인터페이스를 완벽하게 유지함.
