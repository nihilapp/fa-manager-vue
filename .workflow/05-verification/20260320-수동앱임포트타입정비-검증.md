## 1. 설계 대비 구현 대조
- `#imports` 기반 단일 타입 인덱싱 제거:
  - 기대값: 전역 심볼 선언이 실제 모듈 타입을 직접 참조해야 합니다.
  - 구현 결과: `src/app/types/manual-app-imports.d.ts`에 `VueImports`, `PiniaImports`, `NuxtAppImports`, `PiniaNuxtImports` 별칭을 도입하여 각 심볼이 실제 모듈 시그니처를 직접 참조하도록 변경하였습니다.
- Nuxt/Pinia 공개 엔트리포인트 우선 사용:
  - 기대값: 내부 생성 경로 복사 대신 유지보수 가능한 공개 경로를 사용해야 합니다.
  - 구현 결과: Nuxt 관련 전역 선언은 `nuxt/app`, Pinia 관련 선언은 `pinia`, `usePinia`는 `@pinia/nuxt/dist/runtime/composables`를 참조하도록 정리하였습니다.
- 전역 컴포넌트 계약 유지:
  - 기대값: 기존 전역 컴포넌트 선언이 유지되어야 합니다.
  - 구현 결과: `#components` 기반 `GlobalComponents` 확장은 유지하였습니다.

## 2. 태스크 반영 확인
- Task 1: 수동 앱 임포트 선언 재구성 완료
- Task 2: 타입체크 검증 완료

## 3. 테스트 결과
- 수행: `tsc.cmd --noEmit -p tsconfig.json`
  - 결과: 통과
- 수행 시도: `pnpm.cmd exec nuxi typecheck`
  - 결과: 실패
  - 사유: PowerShell 실행 정책 우회 후 재실행했으나 로컬 `vue-tsc` 실행 파일이 없어 npm이 레지스트리 접근을 시도했고, 현재 환경에서 해당 접근이 실패하였습니다.

## 4. 미결 사항 및 위험도
- `nuxi typecheck` 기준 검증은 완료하지 못했습니다.
- 다만 현재 변경 대상이 선언 파일이며 TypeScript 컴파일 기준 오류는 없으므로 위험도는 낮음으로 판단합니다.
- 이후 `vue-tsc`가 로컬에 설치되면 `nuxi typecheck`를 다시 수행하는 것이 바람직합니다.

## 5. 최종 판단
- 수정 반영 가능
- 추가로 `nuxi typecheck` 재검증이 가능하면 더 안전합니다.
