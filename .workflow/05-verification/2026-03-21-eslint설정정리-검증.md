## 설계 준수 여부
- 항목 1:
  - 설계 기대값: TypeScript 공통 규칙을 재사용 가능한 묶음으로 분리
  - 실제 구현: `sharedTsRules` 상수를 추가하고 `.ts`, `.vue` 블록에서 공통 사용
  - 일치 여부: 일치
- 항목 1-1:
  - 설계 기대값: JS/TS 공통 기본 규칙도 `.vue`에서 누락되지 않아야 함
  - 실제 구현: 공통 규칙 블록의 `files` 범위를 `**/*.{js,ts,vue}`로 확장해 Vue SFC에도 동일한 기본 규칙 적용
  - 일치 여부: 일치
- 항목 2:
  - 설계 기대값: `.ts`는 TypeScript 파서를 직접 사용
  - 실제 구현: `files: ['**/*.ts']` 블록에서 `parser: tseslint.parser` 유지 및 공통 TS 규칙 적용
  - 일치 여부: 일치
- 항목 3:
  - 설계 기대값: `.vue`는 Vue 파서를 사용하되 내부 TS는 동일 규칙 적용
  - 실제 구현: `files: ['**/*.vue']` 블록으로 범위를 축소하고 `vue-eslint-parser` + `parserOptions.parser = tseslint.parser` 조합 유지
  - 일치 여부: 일치
- 항목 4:
  - 설계 기대값: Vue 전용 규칙과 import 관련 규칙의 책임 분리
  - 실제 구현: import-x 설정은 공용 블록에 유지하고, Vue 템플릿 및 globals는 `.vue` 전용 블록으로 분리
  - 일치 여부: 일치
