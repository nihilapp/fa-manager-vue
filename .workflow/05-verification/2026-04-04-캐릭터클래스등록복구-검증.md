# 검증 (Verification)

## 요약
- 검증 대상: 캐릭터 클래스 저장/수정/삭제 흐름
- 전체 결과: 수정 완료
- 배포 가능 여부: 가능

## 설계 준수 여부
- 항목 1:
  - 설계 기대값: 서버 클래스 CRUD를 실제 로직으로 복구
  - 실제 구현: POST/PUT/DELETE 엔드포인트에 권한 체크, 존재 확인, DB 작업, 결과 재조회 추가
  - 일치 여부: 일치
- 항목 2:
  - 설계 기대값: 프론트는 클래스명만 있으면 payload 생성
  - 실제 구현: `subClassName` 없이도 payload가 생성되도록 필터 수정
  - 일치 여부: 일치
