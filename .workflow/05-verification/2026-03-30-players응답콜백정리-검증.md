# 검증 (Verification)

## 요약
- 검증 대상: 플레이어 전용 커스텀 훅의 성공 콜백 응답 계약 정리
- 전체 결과: 설계 반영 완료
- 배포 가능 여부: 가능

## 설계 준수 여부
- 항목 1:
  - 설계 기대값: 공통 쿼리 래퍼의 `onSuccess`가 `BaseApiResponse<TData>` 전체를 받는다
  - 실제 구현: `useApiRequest`, `useGet`, `usePost`, `usePut`, `usePatch`, `useDelete` 모두 응답 전체 기준으로 타입 및 전달 로직 수정
  - 일치 여부: 일치
- 항목 2:
  - 설계 기대값: 플레이어 전용 훅은 내부 스토어 갱신에 `response.data`를 쓰고 외부 콜백에는 원본 응답을 전달한다
  - 실제 구현: 목록/상세/내정보/생성/수정 훅 모두 `response.data`로 내부 처리 후 `callback(response)` 호출
  - 일치 여부: 일치
- 항목 3:
  - 설계 기대값: 목록 조회 훅도 단일 응답 객체 기준으로 정리한다
  - 실제 구현: `useGetPlayerList` 콜백이 `BaseApiResponse<ListData<PlayerOutDto>>`를 받도록 수정
  - 일치 여부: 일치
