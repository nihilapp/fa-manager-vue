## 설계 대비 구현 결과
### 헤더 null-safe 렌더링
- 기대값:
  - `myInfo`가 `null`이어도 헤더가 SSR 중 예외 없이 렌더링되어야 한다.
- 실제 구현:
  - `CommonHeader.vue`에서 `displayUserName` 계산값을 사용하고, 값이 있을 때만 이름을 표시하도록 변경했다.
- 판단:
  - 설계 준수

### 사용자 정보 조회 초기화 시점 조정
- 기대값:
  - 사용자 정보 조회가 `onMounted` 전용 흐름이 아니라 앱 초기 진입 시점에서 1회 실행되어야 한다.
- 실제 구현:
  - `app.vue`에서 `callOnce('user-my-info', ...)`를 사용해 `/block` 외 경로에서 `getMyInfo()`를 앱 초기화 중 호출한다.
- 판단:
  - 설계 준수

### 관리자 메뉴 반응성 정리
- 기대값:
  - `isAdmin` 갱신 이후 관리자 메뉴 공개 여부도 다시 계산되어야 한다.
- 실제 구현:
  - `common.store.ts`에서 `navItems`를 `computed`로 전환해 `isAdmin.value`를 반응형으로 읽도록 수정했다.
- 판단:
  - 설계 준수

- 자동 타입체크가 완료되지 않아 SFC 수준 정적 검증이 남아 있다.
- `callOnce`의 실제 런타임 동작은 Nuxt 표준 패턴에 맞지만, 로컬 실행 확인은 아직 수행하지 못했다.
- 위험도: 낮음
- 근거:
  - 변경 범위가 제한적이며, 런타임 예외 원인이던 직접 null 접근은 제거되었다.
