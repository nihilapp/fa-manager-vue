  - 실제 구현: `src/server/types/dto/character.dto.ts`에서 `race?: string`으로 변경했다
  - 일치 여부: 일치
- 항목 2:
  - 설계 기대값: 생성 API는 `name`만 필수로 검사하고 `race`는 빈 문자열로 정규화한다
  - 실제 구현: `src/server/api/characters/index.post.ts`에서 필수 검사에서 `race`를 제외하고 insert 시 `body.race?.trim() ?? ''`를 사용한다
  - 일치 여부: 일치
- 항목 3:
  - 설계 기대값: 생성 폼은 `race` 강제를 제거하고 실패 메시지를 사용자에게 표시한다
  - 실제 구현: `src/app/components/characters/CharacterCreateForm.vue`에서 `race` 강제 payload를 제거했고, API 에러 메시지를 폼 상단 `Box`로 노출한다

### 수동 테스트
- 코드 경로 점검 결과, 생성 폼 submit은 더 이상 `race: ''`를 강제하지 않고 서버가 `race` 미전송을 허용하도록 맞춰졌다

### 통합 테스트
- `pnpm.cmd exec nuxi typecheck` 실행 실패
- 실패 사유: `vue-tsc`를 `https://registry.npmjs.org/vue-tsc`에서 가져오려다 권한/네트워크 제약(`EACCES`)으로 중단됨
- 위험도: 현재 세션에서는 정적 타입 검증을 끝까지 수행하지 못했으므로, 실제 런타임에서 캐릭터 생성 화면 수동 확인이 추가로 필요하다
