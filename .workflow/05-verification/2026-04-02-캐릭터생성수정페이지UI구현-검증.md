## 설계 대비 구현 대조
- 공용 폼 컴포넌트 추가:
  - 기대값: 생성/수정 공용 `CharacterEditorForm`으로 레이아웃을 통합한다.
  - 결과: `src/app/components/characters/CharacterEditorForm.vue`에서 공용 UI를 구현했다.
- 생성 페이지 래퍼 교체:
  - 기대값: `add-character.vue`는 렌더링 컴포넌트만 래핑한다.
  - 결과: 임시 안내 박스를 제거하고 공용 폼 렌더링으로 교체했다.
- 수정 페이지 라우트 추가:
  - 기대값: `edit/[id].vue`가 신규 추가되고 공용 폼을 수정 모드로 렌더링한다.
  - 결과: 동적 라우트 페이지를 추가하고 `characterId`를 전달했다.
- 수정 기본값 주입:
  - 기대값: 기존 캐릭터 조회 결과를 수정 화면 기본값에 반영한다.
  - 결과: `useGetCharacterInfo`와 `characterStore.characterInfo`를 이용해 폼 값을 동기화했다.
- 저장 로직 분리:
  - 기대값: 실제 API 저장 없이 UI와 이벤트 계약만 제공한다.
  - 결과: `submit` 이벤트 방출과 안내 메시지만 제공하고 API 호출은 넣지 않았다.

## 테스트 결과
- 실행: `.\\node_modules\\.bin\\eslint.cmd src\\app\\components\\characters\\CharacterEditorForm.vue src\\app\\pages\\characters\\add-character.vue "src\\app\\pages\\characters\\edit\\[id].vue"`
- 결과: 통과

- 결과: 실패
- 실패 사유: 현재 환경에서 `vue-tsc` 패키지를 레지스트리에서 가져오려다 `EACCES` 및 네트워크 접근 오류가 발생했다.

## 미결 사항
- 실제 생성/수정 API 연결은 아직 구현하지 않았다.
- 클래스 편집 UI는 요약 표시만 제공하며, 개별 클래스 CRUD는 후속 작업이 필요하다.

## 사이드 이펙트 및 위험
- 수정 페이지는 진입 시 캐릭터 스토어의 `characterInfo`를 사용하고 종료 시 초기화한다.
- `typecheck` 미완료 상태이므로 런타임 외 타입 단위 검증은 환경 정상화 후 한 번 더 확인하는 것이 안전하다.
