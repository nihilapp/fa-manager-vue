# 20260320-쿼리훅정리-검증

## 1. 검증 개요
설계 문서 기준으로 query/mutation 훅 인터페이스 통일과 에러 처리 축소 결과를 검증합니다.

## 2. 설계 대조 결과
- [x] `useGet`이 객체 인자 하나를 받도록 변경되었습니다.
- [x] `usePost`, `usePut`, `usePatch`, `useDelete`가 객체 인자 하나를 받고 각 파일에서 직접 `useMutation`을 호출합니다.
- [x] 모든 훅이 `response`와 `execute`를 공통 반환값으로 제공합니다.
- [x] `useApiRequest.ts`는 성공 응답 처리와 Axios 에러 처리 helper만 남겼습니다.
- [x] `api-error-handler.ts`는 미사용 토스트 상태와 `throwIfError`를 제거했습니다.

## 5. 미결 사항 및 위험
- mutation 훅의 실제 소비처가 아직 많지 않아 런타임 사용성은 후속 화면 구현 시 추가 확인이 필요합니다.
- `enabled: false`인 mutation은 `execute` 호출 시 요청을 보내지 않고 `undefined`를 반환하므로, 호출부가 이 동작을 전제로 사용해야 합니다.
