## 설계 대비 구현 결과
### 헤더 우선 식별 및 쿠키 갱신
- 기대값:
  - `x-discord-id` 헤더가 있으면 이를 최우선 식별값으로 사용하고 쿠키를 갱신해야 한다.
- 실제 구현:
  - `resolveDiscordIdentity`가 헤더를 최우선으로 선택한다.
  - 인증 성공 후 `persistDiscordIdCookie`가 `discord_id` 쿠키를 저장한다.
- 판단:
  - 설계 준수

### 쿠키 fallback
- 기대값:
  - 헤더가 없을 때 쿠키 `discord_id`로 사용자를 조회할 수 있어야 한다.
- 실제 구현:
  - 헤더 부재 시 `getCookie(event, 'discord_id')`를 읽어 식별값으로 사용한다.
- 판단:
  - 설계 준수

### stale 쿠키 정리
- 기대값:
  - 쿠키 값으로 사용자 조회가 실패하면 오래된 쿠키를 정리해야 한다.
- 실제 구현:
  - 쿠키 기반 조회 실패 시 `deleteCookie`로 `discord_id`를 제거한다.
- 판단:
  - 설계 준수

### 개발 fallback 유지
- 기대값:
  - 개발 환경에서는 헤더와 쿠키가 모두 없을 때만 `ADMIN_DISCORD_ID`를 마지막 fallback으로 사용해야 한다.
- 실제 구현:
  - `resolveDiscordIdentity`에서 개발 환경 fallback을 마지막 단계로 두고 `source === 'development'`를 유지한다.
- 판단:
  - 설계 준수
