# 설계 (Design)

## 요약
- 변경 대상: `CharacterEditorForm.vue`
- 설계 방향: 축소된 폼 구조는 유지하되 캠페인 선택 드롭다운과 실제 버튼 라벨만 복원한다.
- 주요 결정 사항: `campaignId`는 nullable 선택값으로 관리하고 submit payload에 포함한다.

## 아키텍처 변화
### 수정 구성 요소
- `src/app/components/characters/CharacterEditorForm.vue`
  - `campaignId` 상태 추가
  - `useGetCampaignList` 연동
  - 캠페인 드롭다운 추가
  - 버튼 라벨을 `캐릭터 생성` / `캐릭터 수정`으로 변경

### API / 인터페이스 변경
- submit payload에 `campaignId?: number` 추가

## 로직 흐름
1. 폼 마운트 시 캠페인 목록 요청을 실행한다.
2. `campaignStore.campaignList`를 드롭다운 옵션으로 매핑한다.
3. 수정 모드에서는 `characterInfo.campaignId`를 드롭다운 기본값으로 반영한다.
4. 제출 시 선택된 캠페인이 있으면 `campaignId`를 payload에 포함한다.

## 대안 검토
### 옵션 A
- 설명: 캠페인 ID를 숫자 입력으로 다시 노출
- 장점: 구현이 단순하다.
- 단점: 사용자 실수 가능성이 높고 마스터 요청과 맞지 않는다.
- 미채택 이유: 명시적으로 드롭다운 선택을 요구했다.

### 최종 선택
- 선택 이유: 후속 API 분기 조건을 사용자가 명확히 제어할 수 있고 입력 오류가 줄어든다.
- 기대 효과: 생성/수정 UI가 실제 사용 흐름에 더 가까워진다.
