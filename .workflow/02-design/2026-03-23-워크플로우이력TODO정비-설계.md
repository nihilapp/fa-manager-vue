# 설계 (Design)

## 요약
- 변경 대상: `.workflow/history.md`, `.workflow/TODO.md`, `.workflow/template/TODO.md`, 이번 작업 절차 문서
- 설계 방향: 완료 이력과 진행 현황을 분리하고, `TODO.md`를 세션 재개와 진행 업데이트의 단일 기준 문서로 고정
- 주요 결정 사항: `history`는 최신순 완료 기록만 유지하고, `TODO`는 활성/보류/메모를 한 문서에서 관리

## 아키텍처 변화
### 신규 구성 요소
- `.workflow/TODO.md`: 현재 진행 중 작업 목록, 상태, 다음 행동, 참조 문서를 관리하는 중앙 문서
- `.workflow/template/TODO.md`: 새 작업 시작 시 복제 가능한 진행 현황 템플릿

### 수정 구성 요소
- `.workflow/history.md`: 최신 완료 작업이 상단에 오도록 재정렬하고 이번 완료 작업을 반영
- `.workflow/template/YYYY-MM-DD-작업-보고.md`: 완료 시 `TODO`에서 제거 후 `history`에 올리는 운영 규칙 추가

### DB / 스키마 변경
- 없음

### API / 인터페이스 변경
- 없음

## 로직 흐름
1. 새 작업이 시작되면 분석/설계와 별개로 `.workflow/TODO.md`에 작업 항목을 등록한다.
2. 작업 중에는 상태, 현재 작업, 다음 작업, 참조 문서를 `TODO.md`에서 즉시 갱신한다.
3. 작업이 완료되면 `TODO.md`에서 항목을 제거하거나 완료 섹션으로 이동하지 않고, 바로 `history.md` 상단에 기록한다.
4. 최종 보고 시 `history`와 `TODO`의 상태가 서로 일치하는지 확인한다.

## 대안 검토
### 옵션 A
- 설명: 기존처럼 단계별 문서만 보고 진행 중 작업을 파악한다.
- 장점: 파일 추가가 없다.
- 단점: 재개 시 현재 활성 작업을 한눈에 보기 어렵고 상태 갱신 책임이 분산된다.
- 미채택 이유: 마스터가 요구한 "이 md를 읽어서 현황 파악" 요구를 충족하지 못한다.

### 최종 선택
- 선택 이유: `TODO.md`를 중앙 기준 문서로 두면 세션 재개, 진행 관리, 완료 전환 규칙이 단순해진다.
- 기대 효과: 현재 작업과 완료 작업의 경계가 명확해지고 운영 비용이 줄어든다.

## 제약 및 참고 사항
- `history.md`는 완료 이력만 유지한다.
- `TODO.md`는 진행 중 또는 미해결 작업만 유지한다.
