# 설계 (Design)

## 요약
- 변경 대상: `src/app/pages/characters.vue`, `src/app/pages/characters/index.vue`, 직전 캐릭터 목록 구현 문서
- 설계 방향: `/characters` 엔트리를 `index.vue`로 이동해 도메인 폴더 구조를 기준선으로 삼는다.
- 주요 결정 사항:
  - 라우트 경로는 유지한다.
  - 기능 로직은 바꾸지 않고 파일 위치와 문서 경로 설명만 정리한다.

## 아키텍처 변화
### 신규 구성 요소
- `src/app/pages/characters/index.vue`
  - 기존 캐릭터 목록 페이지 엔트리 역할

### 수정 구성 요소
- `src/app/pages/characters.vue`
  - 삭제
- `.workflow/06-reports/2026-04-02-캐릭터목록페이지구현-보고.md` 등 직전 문서
  - 페이지 경로 설명을 `characters/index.vue` 기준으로 갱신

## 로직 흐름
1. 기존 `characters.vue` 내용을 `characters/index.vue`로 이동한다.
2. 기존 파일은 삭제한다.
3. 관련 문서의 경로 설명을 새 구조로 갱신한다.
4. 변경 파일 기준 ESLint를 다시 수행한다.

## 대안 검토
### 옵션 A
- 설명: 현재 `characters.vue`를 유지하고 나중에 필요할 때만 옮긴다.
- 장점: 당장 수정이 없다.
- 단점: 이미 `characters/` 하위 페이지가 존재해 구조가 혼재된다.
- 미채택 이유: 지금 정리하는 비용이 가장 낮다.

### 최종 선택
- 선택 이유: 도메인 폴더 구조를 빨리 고정하는 편이 이후 상세/수정 페이지 추가에 유리하다.
- 기대 효과: `characters/` 도메인 라우트가 한 폴더 안에서 일관되게 관리된다.
