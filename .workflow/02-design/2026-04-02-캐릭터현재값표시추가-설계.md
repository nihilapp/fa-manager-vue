# 설계 (Design)

## 요약
- 변경 대상: `CharacterEditorForm.vue`
- 설계 방향: 수정 화면에서만 계산 가능한 현재값을 읽기 전용 블록으로 추가한다.
- 주요 결정 사항: 현재 자금은 거래 합산 + 세션 `rewardGold` 합산 결과를 통화 문자열로 표시한다.

## 아키텍처 변화
### 수정 구성 요소
- `src/app/components/characters/CharacterEditorForm.vue`
  - 캐릭터 통화 거래 목록 조회 추가
  - 현재 레벨/현재 자금 계산 computed 추가
  - 읽기 전용 현재값 카드 추가

### API / 인터페이스 변경
- 없음

## 로직 흐름
1. 수정 화면 마운트 시 캐릭터 상세 조회와 함께 캐릭터 통화 거래 목록을 조회한다.
2. 현재 레벨은 `characterInfo.currentLevel`에서 읽는다.
3. 현재 자금은 캐릭터 거래 목록의 `delta*` 합계에 `characterInfo.sessions[*].session.rewardGold` 합계를 GP로 더해 계산한다.
4. 계산 결과를 `CP / SP / EP / GP / PP` 문자열로 렌더링한다.

## 대안 검토
### 옵션 A
- 설명: 서버 API를 수정해 현재 자금을 다시 내려받는다.
- 장점: 프론트 계산이 단순해진다.
- 단점: 현재 요청은 UI 중심 후속 작업이고 서버 수정 범위가 커진다.
- 미채택 이유: 기존 조회 구조만으로 계산 가능하다.

### 최종 선택
- 선택 이유: 기존 상세/거래 조회를 재사용해 빠르게 반영할 수 있다.
