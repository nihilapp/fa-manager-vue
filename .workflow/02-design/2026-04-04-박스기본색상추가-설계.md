# 설계 (Design)

## 요약
- 변경 대상: `Box.vue`의 color 타입과 defaultVariants
- 설계 방향: `Box` 전용 타입에 `default`를 추가하고, `default`를 기본 색상으로 설정한다
- 주요 결정 사항: `StatusColor`는 유지하고 `BoxColor = 'default' | StatusColor` 형태로 국소 확장한다

## 아키텍처 변화
### 신규 구성 요소
- `BoxColor` 지역 타입

### 수정 구성 요소
- `src/app/components/common/Box.vue`
  - `color?: BoxColor`로 변경
  - `variants.color`와 `compoundVariants`에 `default` 추가
  - `defaultVariants.color`를 `default`로 변경

### DB / 스키마 변경
- 없음

### API / 인터페이스 변경
- `Box.color` 허용값이 `default | StatusColor`로 확장된다

## 로직 흐름
1. `Box`가 `color`를 받지 않으면 `default`가 적용된다
2. `default`는 `border-slate-200`만 적용하고 배경색은 추가하지 않는다
3. `gray` 이하 실제 색상 값은 기존처럼 border/background를 함께 적용한다

## 대안 검토
### 옵션 A
- 설명: `StatusColor`에 `default`를 추가한다
- 장점: Box와 다른 컴포넌트가 동일 타입을 그대로 공유한다
- 단점: `default`는 색상 이름이 아니어서 공통 타입 의미가 흐려진다
- 미채택 이유: `StatusColor`는 실제 색상 집합으로 두는 편이 더 명확하다

### 최종 선택
- 선택 이유: `Box` 요구사항만 만족시키면서 공통 타입 의미를 보존할 수 있다
- 기대 효과: `Box`의 기본 표현이 더 예측 가능해지고, 실색상 값과 기본 상태가 분리된다

## 제약 및 참고 사항
- 호출부 수정은 필요 없다. 기본 동작만 달라진다
