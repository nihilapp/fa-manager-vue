# 설계 (Design)

## 요약
- 변경 대상: `src/app/pages/players/index.vue`
- 설계 방향: 테이블의 독자적 배경은 유지하고, 페이지에서는 배경 없는 여백 전용 래퍼만 제공해 과도한 전체 폭 점유를 줄인다
- 주요 결정 사항:
  - 배경이나 카드 스타일은 `DataTable.vue`에 그대로 둔다
  - `players/index.vue`에서만 최대 폭과 가로/세로 패딩을 담당하는 중립 컨테이너를 추가한다
  - 공통 테이블 컴포넌트는 수정하지 않는다

## 아키텍처 변화

  - 래퍼는 `max-width`, `margin-inline: auto`, 적절한 `padding`만 담당

## 로직 흐름
1. 페이지는 기존과 동일하게 목업 플레이어 목록과 pagination 데이터를 계산한다.
2. 템플릿 최상위에서 `DataTable`을 직접 두지 않고, 여백 전용 래퍼 안에 배치한다.
3. 래퍼는 배경 없이 화면 양옆 여백과 최대 폭만 제한한다.
4. `DataTable.vue`는 기존 카드 배경과 shadow를 그대로 사용한다.

## 대안 검토
### 옵션 A
- 설명: `DataTable.vue` 자체에 최대 폭과 margin을 내장
- 장점: 한 번 수정으로 모든 페이지에 동일 적용 가능
- 단점: 공통 컴포넌트가 페이지 레이아웃 책임까지 가져 과도해짐
- 미채택 이유: 폭 제어는 페이지별 요구가 다를 수 있으므로 호출부 책임이 맞다

### 최종 선택
- 선택 이유: 페이지 레이아웃만 조정하면 요구사항을 가장 작게 만족할 수 있다
- 기대 효과: 카드형 테이블은 유지하면서도 화면 가장자리에 숨 쉴 여백이 생긴다

## 제약 및 참고 사항
- 배경이 있는 섹션 래퍼는 다시 도입하지 않는다.
- 프로젝트 Tailwind 클래스 사용 범위 안에서 안전한 폭/여백 클래스만 사용한다.
