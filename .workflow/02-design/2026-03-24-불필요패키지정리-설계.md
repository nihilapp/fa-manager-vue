# 설계 (Design)

## 요약
- 변경 대상: `package.json`, `pnpm-lock.yaml`, `PRD/Task-List.md`
- 설계 방향: 코드에서 이미 제거된 Vue Query/axios 의존성을 패키지 계층과 PRD 계층에서도 제거한다.
- 주요 결정 사항:
  - 제거 대상 패키지는 `@tanstack/vue-query`, `@tanstack/vue-query-devtools`, `axios`
  - 유지 대상은 `@lukemorales/query-key-factory`
  - PRD 문서의 "Vue Query 전환" 표현은 Nuxt 기본 fetcher 기반 표현으로 바꾼다.

## 아키텍처 변화
### 신규 구성 요소
- 없음

### 수정 구성 요소
- `package.json`: 불필요 패키지 제거
- `pnpm-lock.yaml`: 제거 패키지 및 전이 의존성 정리
- `PRD/Task-List.md`: 현재 구현 방향에 맞는 작업 항목으로 수정

### DB / 스키마 변경
- 없음

### API / 인터페이스 변경
- 없음

## 로직 흐름
1. 코드/설정에서 Vue Query와 axios 잔존 참조가 없는지 재확인한다.
2. `pnpm remove`로 대상 패키지를 제거하고 lockfile을 갱신한다.
3. `PRD/Task-List.md`의 오래된 Vue Query 관련 작업 문구를 현재 fetcher 방향에 맞게 수정한다.
4. 전역 검색으로 잔존 참조 여부를 검증한다.

## 대안 검토
### 옵션 A
- 설명: 코드만 제거하고 패키지 의존성은 남겨 둔다.
- 장점:
  - 변경 폭이 작다.
- 단점:
  - 죽은 의존성이 계속 남아 기준선이 흐려진다.
- 미채택 이유:
  - 마스터 요청에 부합하지 않는다.

### 최종 선택
- 선택 이유:
  - 코드/설정/패키지/문서 기준선을 모두 일치시킬 수 있다.
- 기대 효과:
  - 설치 대상이 가벼워지고, 프로젝트 방향이 명확해진다.

## 제약 및 참고 사항
- lockfile 갱신은 권한 상승이 필요했다.
- `pnpm remove` 결과로 일부 패키지 버전 표기가 최신 값으로 재정렬될 수 있다.
