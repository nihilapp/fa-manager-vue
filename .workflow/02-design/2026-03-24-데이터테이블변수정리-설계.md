# 설계 (Design)

## 요약
- 변경 대상: `src/app/components/table/DataTable.vue`
- 설계 방향: 페이지네이션 정규화 결과를 단일 기준 상태로 유지하고, `Paginator.vue`에 명시적 props/emit 계약을 부여해 `DataTable.vue`의 중복 변수를 제거한다.
- 주요 결정 사항:
  - `Paginator.vue`의 암묵적 상위 스코프 의존을 제거한다.
  - `DataTable.vue`는 `normalizedPagination`, `paginationEnabled`, `normalizedPageButtonCount` 중심으로 축소한다.
  - 페이지 버튼 계산과 페이지 이동 UI 로직은 `Paginator.vue`로 이동한다.

## 아키텍처 변화
### 신규 구성 요소
- 없음

### 수정 구성 요소
- `src/app/components/table/DataTable.vue`
  - `normalizedPagination`을 페이지네이션의 단일 기준 상태로 유지한다.
  - `Paginator.vue`에 필요한 데이터를 props로 전달하고, `pageChange` 이벤트만 수신한다.
  - `filteredElements`, `currentPage`, `hasPrev`, `hasNext`, `pageNumbers`, `startItem`, `endItem`, 페이지 이동 함수군 등 페이지네이터 전용 상태를 제거한다.
- `src/app/components/table/Paginator.vue`
  - 상위 스코프 식별자를 직접 참조하던 구조를 props/emit 기반 독립 컴포넌트로 바꾼다.
  - 전달받은 `pagination`, `itemsLength`, `pageButtonCount`를 기준으로 표시 수치와 페이지 이동 버튼을 계산한다.

### DB / 스키마 변경
- 없음

### API / 인터페이스 변경
- 없음
- `DataTable.vue` props/emit 계약 유지
- `Paginator.vue` 내부 계약 변경:
  - props: `pagination`, `itemsLength`, `pageButtonCount`
  - emit: `pageChange`

## 로직 흐름
1. `props.pagination`을 `normalizedPagination`에서 안전한 구조로 정규화한다.
2. `DataTable.vue`는 페이지네이터용 세부 파생 상태를 계산하지 않고, 정규화 결과와 최소 부가 정보만 `Paginator.vue`에 전달한다.
3. `Paginator.vue`는 전달받은 props로 `startItem`, `endItem`, `pageNumbers`, 이전/다음 버튼 상태를 자체 계산한다.
4. `Paginator.vue`의 버튼 클릭은 `pageChange` 이벤트를 발생시키고, `DataTable.vue`는 이를 상위로 재전달한다.
5. 렌더링 결과와 외부 페이지 변경 계약은 유지하면서 `DataTable.vue`의 선언 수를 줄인다.

## 대안 검토
### 옵션 A
- 설명: `Paginator.vue`를 props 기반 독립 컴포넌트로 재설계하고, `DataTable.vue`의 페이지네이션 기준 상태만 명시적으로 전달한다.
- 장점:
  - 상위 스코프 암묵 의존성이 사라진다.
  - 실제 사용값과 인터페이스가 분리되어 구조가 명확해진다.
  - `DataTable.vue`의 미사용 변수 경고를 구조적으로 해소할 수 있다.
- 단점:
  - `Paginator.vue`까지 수정 범위가 확대된다.

### 옵션 B
- 설명: 지금 선언된 computed를 모두 유지하고 이름만 정리한다.
- 장점:
  - 위험이 가장 낮다.
- 단점:
  - 불필요한 중복 선언이 남아 요청 목적을 거의 달성하지 못한다.
- 미채택 이유:
  - "굳이 필요 없는 변수는 없애고"라는 요청에 충분히 부합하지 않는다.

### 최종 선택
- 선택 이유:
  - 외부 계약은 유지하면서 `DataTable.vue`의 실제 불필요 변수를 가장 많이 제거할 수 있다.
  - 정적 검사에서 드러난 암묵적 상위 스코프 의존 문제를 함께 해소할 수 있다.
- 기대 효과:
  - `DataTable.vue` 스크립트가 짧아지고, 페이지네이터의 책임이 `Paginator.vue`로 명확히 이동한다.

## 제약 및 참고 사항
- `DataTable.vue`의 외부 props/emit 계약은 유지한다.
- `Paginator.vue` 내부 계약은 이번 작업에서 명시적으로 재정의한다.
- `context7 MCP`는 현재 세션에서 사용할 수 없어 로컬 코드 기준으로 설계했다.
