## 활성 태스크 수행 기록
### Task 1: 헤더 사용자 정보 null-safe 렌더링 적용
- 수정 파일:
  - `src/app/components/layout/common-layout/CommonHeader.vue`
- 수행 내용:
  - `myInfo!.name` 직접 접근을 제거했다.
  - `displayUserName` 계산값을 도입하고 값이 있을 때만 사용자 이름을 렌더링하도록 변경했다.
- 판단 근거:
  - `myInfo === null`은 스토어 계약상 허용된 초기 상태이므로 템플릿에서 이를 오류로 취급하면 안 된다.

### Task 2: 사용자 정보 조회 초기화 시점 조정
- 수정 파일:
  - `src/app/app.vue`
  - `src/app/stores/user.store.ts`
- 수행 내용:
  - `onMounted` 기반 초기화를 제거하고 `callOnce('user-my-info', ...)`로 앱 초기 진입 시점 1회 호출로 변경했다.
  - `getMyInfo()` 내부 디버그 로그를 제거했다.
- 판단 근거:
  - SSR 시점에도 사용자 정보 조회가 가능한 구조로 이동해야 초기 렌더와 하이드레이션 간 불일치를 줄일 수 있다.

### Task 3: 관리자 메뉴 반응성 정리 및 최소 검증
- 수정 파일:
  - `src/app/stores/common.store.ts`
- 수행 내용:
  - `navItems`를 `ref`에서 `computed`로 전환해 `isAdmin` 변경이 메뉴 공개 여부에 반영되도록 수정했다.
  - 코드 대조와 타입체크 시도를 통해 검증 준비 상태를 정리했다.
- 판단 근거:
  - `isAdmin.value`를 초기값으로 고정하면 사용자 정보 로딩 이후 관리자 메뉴 표시가 갱신되지 않을 수 있다.

## 임시 검증 결과
- 코드 대조 결과:
  - 헤더의 직접 null 접근 제거 확인
  - 앱 초기화 시점 이동 확인
  - 관리자 메뉴 공개 조건 반응형 계산 전환 확인
- 명령 검증:
  - `pnpm exec nuxi typecheck`: PowerShell 실행 정책으로 실패
  - `pnpm.cmd exec nuxi typecheck`: `vue-tsc` 패키지 조회가 네트워크 제약으로 실패
