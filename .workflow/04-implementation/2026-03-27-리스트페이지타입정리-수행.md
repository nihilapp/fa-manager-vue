## 판단 근거
- `ListPageData`는 서버 응답 타입의 파생 형태라 서버 타입 파일에 두는 편이 구조상 맞다.
- 비페이징에서 `null`보다 `0/false`를 내려주면 프런트 로직이 단순해진다.
- 페이지 이동 가능 여부는 `prevPage/nextPage` 값보다 `hasPrev/hasNext`가 기준이 되는 편이 안전하다.

## 임시 검증 결과
- `ListDataType`의 페이지 nullable 타입을 제거했다.
- `DataTable`과 `Paginator`는 더 이상 페이지 `null`을 전제로 하지 않는다.
