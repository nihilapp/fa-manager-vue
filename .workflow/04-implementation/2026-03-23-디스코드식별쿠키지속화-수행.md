## 활성 태스크 수행 기록
### Task 1: 서버 식별값 우선순위 및 쿠키 유틸 정리
- 수정 파일:
  - `src/server/utils/auth.ts`
- 수행 내용:
  - `discord_id` 쿠키 키와 쿠키 옵션 상수를 추가했다.
  - 헤더, 쿠키, 개발 fallback 순으로 식별값을 해석하는 `resolveDiscordIdentity`를 추가했다.
- 판단 근거:
  - 공통 인증 경로에서 식별 우선순위를 일관되게 다뤄야 API별 편차가 생기지 않는다.

### Task 2: `authHelper` 쿠키 지속화 구현
- 수정 파일:
  - `src/server/utils/auth.ts`
- 수행 내용:
  - 헤더 기반 식별 성공 시 `discord_id` 쿠키를 저장하도록 구현했다.
  - 헤더가 없으면 쿠키 값을 사용해 사용자 조회를 수행하도록 변경했다.
  - 쿠키 기반 조회 실패 시 stale 쿠키를 삭제하도록 구현했다.
- 판단 근거:
  - 최초 진입 헤더는 후속 요청에 자동 전달되지 않으므로 서버가 식별값을 지속화해야 한다.

### Task 3: 인증 흐름 영향 검증 및 문서화
- 수정 파일:
  - `.workflow` 문서 일체
- 수행 내용:
  - `/users/me`를 포함해 `authHelper(event)`를 사용하는 API 전체가 같은 쿠키 fallback 규칙을 공유하게 됨을 코드 기준으로 검토했다.
  - 검증과 보고를 위한 산출물을 정리했다.
- 판단 근거:
  - `authHelper` 단일 진입점 변경으로 공통 인증 정책이 전체 API에 전파된다.

## 임시 검증 결과
- 코드 대조 결과:
  - `x-discord-id` 헤더 우선 해석 확인
  - `discord_id` 쿠키 fallback 확인
  - stale 쿠키 삭제 확인
  - 개발 환경 `ADMIN_DISCORD_ID` 마지막 fallback 유지 확인
- 자동 실행 검증:
  - 별도 서버 실행 검증은 수행하지 못함
