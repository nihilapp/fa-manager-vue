## 구현 메모
- 수정 파일:
  - `src/app/pages/players/index.vue`
  - `src/app/components/table/DataTable.vue`
- 적용 내용:
  - 플레이어 목록 페이지의 실제 API 조회를 제거하고 25건 목업 데이터 및 로컬 pagination 계산으로 전환
  - `@page-change`를 로컬 `currentPage`와 연결해 페이지 이동 가능하게 조정
  - 바깥 `section` 래퍼와 배경 제거
  - `DataTable.vue`에서 `pageSize` 기준 placeholder row를 계산해 마지막 페이지에서도 고정된 행 수를 유지
  - 전체 데이터가 0건일 때만 empty state가 보이도록 조건 분리
- 판단 근거:
  - 시각 테스트와 페이징 UX 확인이 목적이므로 페이지 내부 목업 데이터가 가장 빠르고 안전함
  - 빈 행 보강은 개별 페이지가 아니라 공통 테이블이 책임지는 편이 재사용성과 일관성이 높음
