## 판단 근거

- 프런트는 이미 `/players` 라우트를 사용하고 있었으므로 내부 식별자와 API 호출만 새 명칭으로 맞추면 된다.
- 서버는 기존 `users` API 디렉터리를 `players`로 이동시키고, DTO/테이블 상수도 같은 이름 체계로 정렬해야 API 계약 변경이 성립한다.
- 응답 메시지는 플레이어 상수를 추가하고 기존 사용자 상수는 호환 별칭으로 남겨 연쇄 파손을 줄였다.

## 임시 검증 결과

- `rg` 기준으로 `src/app`, `src/server`, `PRD` 내부의 `/users` 직접 경로 참조는 제거되었다.
- `pnpm.cmd exec nuxi typecheck`는 로컬에서 `vue-tsc`를 레지스트리에서 가져오려다 실패하여 완료하지 못했다.
- DB 실제 마이그레이션과 relation 필드명(`user`, `userId`) 전면 변경은 이번 수정에서 일부 호환 형태로 남아 있어 후속 점검이 필요하다.
