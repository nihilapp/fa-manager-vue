## 구현 메모
- 공통 렌더링 컴포넌트 `PlayerProfileDetail.vue`를 추가해 플레이어 기본 정보와 캐릭터/캠페인/세션 리스트 섹션을 props 기반으로 렌더링하도록 정리했다.
- `me/index.vue`는 `useGetMyInfo`, `useGetMyCharacterList`, `useGetMyCampaignList`, `useGetMySessionList`를 실행해 같은 공통 뷰를 사용하도록 바꿨다.
- `players/detail/[id].vue`는 기존 단건/캠페인/세션 조회에 더해 `useGetCharacterList({ userId })`를 연결하고 공통 뷰를 사용하도록 바꿨다.
- 기존 `PlayerDetail.vue`는 호출부 호환을 위해 공통 뷰를 감싸는 래퍼 형태로 단순화했다.
- 실제 라우트가 없는 상세 경로 링크는 현재 존재하는 목록 페이지 경로(`/characters`, `/campaigns`, `/sessions`)로 맞췄다.
