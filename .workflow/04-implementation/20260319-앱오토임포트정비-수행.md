## 판단 근거
- 템플릿 폴더가 없어 기존 단계 산출물 형식을 기준으로 신규 문서를 직접 작성합니다.
- 앱 쿼리 컴포저블은 중복이 심해 공통 헬퍼 `useApiRequest.ts`로 인증 헤더 생성과 성공/실패 처리 규약을 통합했습니다.
- `useGet`은 기존 `enabled` 감시와 `useFetch.watch`가 중복 실행될 수 있어 수동 watch 기반 단일 진입으로 정리했습니다.
- `src/app`의 `~/` 경로 참조는 실제 파일 구조와 맞지 않아 `@app`, `@server` 기준으로 정리했습니다.
- 앱 오토임포트는 `src/server`와 같은 선언형 방식으로 `nuxt.config.ts`에 명시 등록했습니다.
- 프론트 응답 타입은 앱 자체 별칭 대신 서버 전역 타입 `BaseResponse`, `ListData`를 직접 사용하도록 정리했습니다.
- 추가 기준선으로 이벤트 핸들러 네이밍은 `on<Action><Target>` 형식으로 `PRD/Coding-Rules.md`에 명시했습니다.

## 임시 검증 계획
- 타입 정의와 쿼리 컴포저블 수정 후 `nuxt typecheck`로 정합성을 점검합니다.

## 임시 검증 결과
- `pnpm.cmd exec nuxt typecheck`는 `vue-tsc`를 외부 레지스트리에서 가져오려다 권한/네트워크 제약으로 실패했습니다.
- 대체 검증으로 `pnpm.cmd exec tsc --noEmit`를 실행했고 현재 수정 범위 기준 타입 오류 없이 통과했습니다.
