## 구현 메모
- 수정 파일:
  - `src/app/pages/players/index.vue`
  - `src/app/components/table/DataTable.vue`
  - `src/app/components/table/Paginator.vue`
- 적용 내용:
  - 플레이어 목록 액션을 공통 `Button.vue` 기반으로 교체
  - `useUserStore()`의 `isAdmin`을 사용해 수정 버튼을 관리자에게만 노출
  - 상세보기/수정 버튼은 현재 모두 `/players/[id]` 경로로 이동하도록 정리
  - 상태 값을 배지 형태로 표현하도록 플레이어 페이지 슬롯 스타일 조정
  - `DataTable.vue`를 밝은 surface, 얇은 경계, hover 강조 중심의 데이터 그리드 톤으로 재구성
  - `Paginator.vue`를 pill형 버튼 그룹과 정보 라벨 구조로 정리
- 판단 근거:
  - 버튼 컴포넌트 사용 규칙을 플레이어 페이지에 실제 적용해 일관성을 확보하기 위함
  - 에어테이블 느낌은 개별 셀보다 테이블/페이지네이터 전체 시각 언어를 같이 바꿔야 자연스럽기 때문
