# 20260320-뷰쿼리전환-초기화

## 1. 개요
현재 Nuxt의 `useFetch` 및 `$fetch` 기반의 데이터 패칭 구조를 `@tanstack/vue-query`로 전환하고, 쿼리 키 빌더를 도입하여 캐시 관리를 체계화한다.

## 2. 배경 및 목적
- **캐시 관리 최적화**: Nuxt `useFetch`보다 정밀한 캐시 제어(staleTime, cacheTime 등) 필요.
- **상태 관리 효율화**: Pinia에 직접 API 결과를 저장하는 방식에서 Query Cache를 참조하는 방식으로 전환하여 데이터 일관성 확보.
- **코드 일관성**: 쿼리 키 빌더를 도입하여 하드코딩된 문자열 기반 캐시 키 관리 문제를 해결.

## 3. 작업 범위
- 패키지 설치: `@tanstack/vue-query`, `@lukemorales/query-key-factory`
- Nuxt 플러그인 등록 및 설정 (SSR 대응)
- 커스텀 컴포저블(`useGet`, `usePost` 등) 내부 로직 리팩토링
- 쿼리 키 팩토리 정의 및 적용
- 기존 `user.store.ts` 연동 코드 수정

## 4. 판단 근거
- 현재 프로젝트 초기 단계로 리팩토링 비용이 매우 낮음.
- 향후 복잡한 데이터 동기화(캐시 무효화 등) 요구사항에 대응하기 위해 선제적 도입 결정.
- 마스터의 `Vue Query` 사용 결정 및 기존 인터페이스 유지 지침 준수.
