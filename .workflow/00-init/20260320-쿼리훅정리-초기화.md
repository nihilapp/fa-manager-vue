# 20260320-쿼리훅정리-초기화

## 1. 개요
`src/app/composables/query` 계층의 훅 시그니처를 정리하고, `src/app/utils/api-error-handler.ts`의 불필요한 요소를 축소하는 작업의 기준선을 확립합니다.

## 2. PRD 및 작업 구조 점검
- `PRD/PRD.md`, `PRD/Task-List.md`, `PRD/Coding-Rules.md`가 모두 존재하여 이번 작업은 초기화가 아니라 갱신으로 판단합니다.
- `.workflow/00-init`부터 `.workflow/06-reports`까지 절차 폴더가 존재하여 기록 구조는 유지 가능한 상태입니다.
- `.workflow/template` 경로는 현재 저장소에 존재하지 않아, 기존 `.workflow` 산출물 형식을 참조하여 동일한 단계 문서를 작성합니다.

## 3. 이번 작업의 기준
- 요구사항 기준: `useGet`과 `usePost` 계열 훅의 인자를 객체 하나로 통일하고, mutation 훅은 각 파일에서 직접 `useMutation`을 호출합니다.
- 구현 규칙 기준: 공통 에러 처리는 유지하되 실제 사용되지 않는 API와 상태는 제거합니다.
- 검증 기준: 타입 검사 통과와 기존 사용처(`user.store.ts`)의 호출 방식 정합성 확보를 완료 조건으로 삼습니다.

## 4. 생성 및 갱신 대상
- 코드 대상: `src/app/composables/query/*.ts`, `src/app/utils/api-error-handler.ts`, `src/app/stores/user.store.ts`
- 절차 문서 대상: `.workflow/00-init`부터 `.workflow/06-reports`까지 본 작업 기록
