# Product Requirements Document (PRD)

## 1. 개요
- **목표**: TRPG 캠페인, 세션, 캐릭터, 플레이어 관리를 용이하게 하기 위한 보조 관리 툴 개발
- **주요 가치**: 다양한 로그(경험치/소비현황 등) 추적 및 편리한 다중 캠페인/캐릭터 관리 제공

## 2. 프로젝트 특성 구상
- **사용자 식별**: 
  - 정통적인 비밀번호 기반 회원가입/로그인은 생략 (초안).
  - 디스코드(Discord) 봇을 통해 접근하거나, 웹 UI 상에서 디스코드 사용자 고유 ID(숫자)를 입력하여 접속/식별 처리 예정 (변동 가능성 높음).
- **다중 사용자 지원**: 여러 명의 사용자가 동시에 서비스를 활용 가능.

## 3. 핵심 도메인 및 엔티티 관계
- **주요 도메인**: 캠페인(Campaign), 세션(Session), 캐릭터(PC), 플레이어(User)
- **관계성**:
  - 사용자 <-> 캠페인: 사용자는 여러 캠페인을 생성하거나 기존 캠페인에 가입할 수 있다 (N:M 혹은 1:N 멤버 매핑).
  - 사용자 -> 캐릭터(PC): 사용자는 캠페인에 소속된 새로운 캐릭터(PC)를 다수 생성할 수 있다.
  - 캠페인 -> 세션: 하나의 캠페인에는 진행된 여러 세션(Session)이 속성으로 기록된다.
  - 캠페인 -> 캐릭터(PC): 캠페인마다 여러 명의 캐릭터(PC)들이 종속된다.

## 4. 상세 사양 (Functional Requirements)
- **캠페인 관리 모듈**:
  - 캠페인 방 생성 기능.
  - 다른 사용자의 캠페인에 대한 가입(참여) 기능.
- **캐릭터(PC) 모듈**:
  - 특정 캠페인 소속의 내 캐릭터(PC) 생성 및 프로필 유지 기능.
- **세션 모듈**:
  - 캠페인 마스터 또는 권한자가 새로운 세션 기록을 등록/수정.
- **로그 및 추적 시스템 (Logs & Tracking)**:
  - **경험치 추적 로그**: 캐릭터 경험치 변동 시 변경 전후 및 사유가 기록된다.
  - **소비현황 추적 로그**: 자원(소지금, 아이템 리소스 등) 변경 시 추적 기록이 남는다.
  - **특징**: 데이터 변경 행위 시 자동으로 트리거되어 이력이 누적되어야 함.

## 5. 핵심 기술 스택
- **프레임워크**: Nuxt 3 (별도 외부 API 서버 없이 통합 개발)
- **데이터베이스 ORM**: Drizzle ORM
- **언어 및 환경**: TypeScript, Node.js 생태계
